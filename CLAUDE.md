# guardrail3 — Agent Instructions

> **Agent-managed codebase.** Do not say "you can edit X" — just do it.

## What This Tool Does

Rust CLI that validates and generates code guardrails for Rust and TypeScript projects. Single binary, no Node.js dependency.

## Commands

| Command | What it does |
|---|---|
| `guardrail3 validate [path]` | Auto-detect stacks, run all checks |
| `guardrail3 rs validate [path]` | Rust checks only (R1-R57) |
| `guardrail3 ts validate [path]` | TypeScript checks only (T1-T61) |
| `guardrail3 hooks validate [path]` | Hook + deployment checks (H1-H11, D1-D5) |
| `guardrail3 init --profile <name>` | Scaffold guardrail3.toml + local/ |
| `guardrail3 generate` | Produce config files from guardrail3.toml |
| `guardrail3 check` | CI: verify generated files are current |
| `guardrail3 diff` | Dry run of generate with diffs |
| `guardrail3 list-modules` | List embedded modules |
| `guardrail3 show-module <name>` | Print module content |

## Project Structure

```
src/
  main.rs, cli.rs              — entry point, clap CLI
  discover.rs                  — auto-detect Rust/TS stacks
  config/                      — guardrail3.toml parser
  commands/                    — init, generate, check, diff, validate, modules
  modules/                     — embedded guardrail content (clippy, deny, canonical, pre-commit)
  rs/validate/                 — Rust validation (57 checks across 10 modules)
  ts/validate/                 — TypeScript validation (61 checks across 7 modules)
  hooks/                       — Hook + deployment validation (16 checks)
  report/                      — text, json, markdown formatters
```

## Guardrails (self-enforced)

This project uses profile `minimal` with its own guardrail3 config. Run `guardrail3 validate .` to check compliance.

- clippy.toml, deny.toml, rustfmt.toml, rust-toolchain.toml — generated by `guardrail3 generate`
- Cargo.toml has full workspace lint configuration
- Max 500 effective lines per file
- No `#[allow]` without justification comment
- No `.unwrap()` — use proper error handling
- No `unsafe` — forbidden at workspace level
- No `HashMap`/`HashSet` — use BTree variants

## Adding a New Check

1. Add the check function to the appropriate validate module (rs/ts/hooks)
2. Return `Vec<CheckResult>` with correct ID, severity, title, message
3. Wire it into the module's `check()` orchestrator
4. Keep files under 500 lines — split into sub-modules if needed

## Adding a New Module

1. Add const in `src/modules/clippy.rs` or `src/modules/deny.rs`
2. Register in the profile functions (service, library, minimal)
3. Register in `src/modules/mod.rs` `all_modules()` function

## Don'ts

1. Don't use `.unwrap()` — use `?`, `if let`, or `match`
2. Don't use `unsafe` — forbidden
3. Don't use `HashMap`/`HashSet` — use `BTreeMap`/`BTreeSet`
4. Don't create files over 500 effective lines — split into modules
5. Don't add `#[allow]` without a `//` justification comment
6. Don't relax guardrail configs without `// EXCEPTION: reason`
