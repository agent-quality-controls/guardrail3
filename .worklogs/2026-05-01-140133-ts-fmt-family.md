## Summary

Added the `fmt` TypeScript family to G3TS. The family delegates formatting to Prettier and only enforces package, config, script, Syncpack, hook, and CLI wiring.

## Decisions Made

- `g3ts-fmt` checks wiring, not source formatting. Prettier owns actual formatting.
- Prettier config files are presence-only in this pass. G3TS does not parse Prettier config without a dedicated shared parser package.
- `format:check` fail-closed behavior is a separate semantic rule: `g3ts-fmt/format-check-fail-closed`.
- `g3ts-fmt/policy-configured` fails when no package root can be evaluated, including a standalone Prettier config without `package.json`.
- Config checks use one semantic module per rule and rule-specific sidecar tests where implemented.
