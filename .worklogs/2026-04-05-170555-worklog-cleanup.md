# Clean Up Historical Worklogs

**Date:** 2026-04-05 17:05
**Scope:** `.worklogs/2026-04-04-150600-session3-handoff.md`, `.worklogs/2026-04-03-231827-deny-toml.md`, `.worklogs/2026-04-05-125907-deny-toml-parser-full-schema-handoff.md`, `.worklogs/2026-04-05-132628-parser-schema-audit-handoff.md`

## Summary
Cleaned up the remaining dirty worklog state after the extraction and parser passes. The old session handoff was rewritten into an explicit historical note, the older deny parser worklog was updated so it no longer points at pre-rename package paths, and the two parser audit handoffs were kept as repo history because they contain useful schema and boundary context.

## Context & Problem
After the cargo and deny extraction commits, the remaining dirty files were all worklogs and handoff notes. Some of them were stale enough to be misleading: the session-3 handoff still looked like a current design memo even though the architecture had since been resolved and implemented, and the older deny parser note still referred to the pre-rename `deny-toml` package path.

At the same time, two untracked parser audit handoffs existed and were worth preserving because they explain why the parser packages were broadened and what “file-faithful parser” means in this repo.

## Decisions Made

### Convert the old session handoff into a historical note
- **Chose:** Rewrite `.worklogs/2026-04-04-150600-session3-handoff.md` into a short historical handoff that explicitly says it is no longer the source of truth.
- **Why:** The old file mixed useful architectural turning points with stale arguments and outdated names. Keeping it verbatim would mislead future agents.
- **Alternatives considered:**
  - Leave it untouched — rejected because it still implied unresolved design debates that are already settled.
  - Delete it — rejected because it still captures the important moment when extraction became a strict orchestrator/parsed-file/content-package split.

### Preserve older deny parser history, but fix the rename drift
- **Chose:** Keep `.worklogs/2026-04-03-231827-deny-toml.md` and add a historical note clarifying that the package is now `deny-toml-parser`.
- **Why:** The worklog still contains real historical design context about the original parser build, but its paths were stale after the parser-package rename.
- **Alternatives considered:**
  - Drop the file entirely — rejected because it is still useful provenance.
  - Rewrite the whole file into a new style — rejected because the main issue was path/name drift, not missing context.

### Keep the parser audit handoffs as repo history
- **Chose:** Commit `.worklogs/2026-04-05-125907-deny-toml-parser-full-schema-handoff.md` and `.worklogs/2026-04-05-132628-parser-schema-audit-handoff.md`.
- **Why:** They explain the parser broadening work against upstream docs and capture the “model the file, not the current consumer” rule that now governs the parser packages.
- **Alternatives considered:**
  - Treat them as temporary local notes — rejected because they provide durable parser-package context for future extraction work.

## Architectural Notes
This cleanup reinforces the current repo boundary:

```text
app family orchestrator
  -> discovers authoritative files
  -> parses files
  -> emits structural/parse failures
  -> calls content-check package with parsed file types
  -> package validates content only
```

The cleaned handoff now points readers at the current extraction ledger instead of preserving outdated debate. The parser handoffs remain because the parser packages are an important upstream dependency for the extracted content-check packages.
