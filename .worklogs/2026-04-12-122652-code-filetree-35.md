# Code File-Tree 35

## Summary

Built the missing `code` file-tree lane in packages and migrated old app `RS-CODE-35` into `RS-CODE-FILETREE-35`. The new lane now measures structural caps per owned package root, wires through `g3rs-code-ingestion`, and has direct rule tests plus end-to-end pipeline coverage for thresholds, workspace members, deepest-root ownership, and `target` exclusion.

## Decisions made

- Added a dedicated `g3rs-code-file-tree-checks` package instead of widening config or source.
  - Why: `RS-CODE-35` checks structural tree shape, not config contents or source contents.
- Reused `config_scope` root ownership for package-root selection.
  - Why: the file-tree lane should use the same workspace-member surface as the existing `code` config lane.
- Assigned `.rs` files to the deepest selected package root before measuring caps.
  - Why: root packages must not absorb member package structure.
- Kept the old rule thresholds and diagnostic wording.
  - Why: this is a migration of one live rule, not a policy change.
- Removed the stale `FileTreeIngestionNotImplemented` error variant.
  - Why: leaving it in the public API after implementing the lane would be a lie.
