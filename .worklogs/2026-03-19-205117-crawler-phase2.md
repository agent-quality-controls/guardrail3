# Wire crawler into validate — Phase 2: replace hardcoded paths

**Date:** 2026-03-19 20:51
**Scope:** 12 files across rs/validate, ts/validate, hooks

## Summary
Replaced all hardcoded path construction with crawler data. Every check module now receives config file paths from the crawler instead of constructing them via `workspace_root.join()`.

## RS changes
- mod.rs: filters crawler vecs by workspace root, passes down
- config_files.rs: uses crawl.clippy_tomls, rustfmt_tomls, rust_toolchains
- clippy_coverage.rs: uses crawl.clippy_tomls
- deny_audit.rs: iterates ALL deny.tomls within workspace (multi-config support)
- cargo_lints.rs: uses crawl.cargo_tomls

## TS changes
- mod.rs: passes crawler fields to all check modules
- eslint_check.rs: iterates crawl.eslint_configs
- tsconfig_check.rs: uses crawl.tsconfigs + tsconfig_bases
- npmrc_check.rs: iterates crawl.npmrcs
- package_check.rs: uses crawl.package_jsons, root identified by parent match
- jscpd_check.rs: uses crawl.jscpd_configs + velite_configs
- hooks: uses crawl.pre_commit_hooks
