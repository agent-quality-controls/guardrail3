# Fix hexarch test regression: 171 → 5 failures

**Date:** 2026-04-02 19:07

## Root cause
Hexarch's map_rs_hexarch used legal_workspace_roots which only returns
top-level workspace roots. App roots (apps/backend etc.) are nested
workspaces under the project root — legality marks them as illegal
(RS-TOPOLOGY-11: no nested workspaces). So hexarch saw zero app roots.

## Fix
Changed map_rs_hexarch to use map_global_roots_for_family (all placement
roots) instead of map_workspace_roots_for_family (only legal workspace roots).
Hexarch needs ALL app-classified roots regardless of workspace nesting legality.

## Remaining 5 failures
- rs_hexarch_21 domain purity (2): likely content/scoping differences
- rs_hexarch_14 dependency inventory (1): exact message mismatch
- rs_hexarch_11 root workspace (2): root Cargo.toml content access
