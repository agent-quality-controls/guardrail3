# Session Handoff — March 16-17, 2026

## What guardrail3 IS

A Rust CLI tool (`cargo install guardrail3`) that validates code project configuration and architecture. It does NOT re-implement linters. It ensures that linters, build tools, and architectural patterns are correctly configured so THEY catch violations at compile time or pre-commit.

**The principle:** guardrail3 validates the guardrails themselves. Clippy catches unwrap. guardrail3 checks that clippy IS CONFIGURED to catch unwrap. cargo-deny catches banned crates. guardrail3 checks that deny.toml exists and is correct. The pre-commit hook runs all tools. guardrail3 checks the hook is configured.

### Philosophy (from CLAUDE.md)

guardrail3 is NOT a linter. It is a **configuration and architecture enforcer**:
1. Verifies linter rules ARE configured (clippy lints, ESLint rules, deny.toml bans)
2. Validates project structure (hex arch, dependency flow)
3. Audits every escape hatch (every #[allow] needs a reason)
4. Enforces dependency boundaries (libraries can't use I/O crates)
5. Checks input validation exists at trust boundaries (garde/Validate)

It does NOT scan source for what clippy/ESLint already catch. R26 verifies clippy is configured. R45 verifies cargo-deny is installed. The pre-commit hook runs them. guardrail3 ensures the guardrails themselves are in place.
