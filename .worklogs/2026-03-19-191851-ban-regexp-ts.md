# Ban RegExp in TypeScript via ESLint guardrail

**Date:** 2026-03-19 19:18
**Scope:** eslint_check.rs, canonical.rs

## Summary
Added T-ESLP-15 check requiring ESLint config to ban RegExp via no-restricted-globals and regex literals via no-restricted-syntax. Updated canonical ESLint module with the ban. Verified neither steady-parent nor ts-rust-railway uses RegExp anywhere — Zod handles all validation.

## Note on .contains()
Cannot ban globally — it's str::contains, Vec::contains, BTreeSet::contains, all legitimate. The architectural enforcement (CLAUDE.md principle 7 + parser migration) is the correct approach for eliminating .contains() on raw file content.
