# Split 27 deny rule mod.rs into rule.rs + tests/helpers.rs

Split all 27 unsplit deny rules (rs_deny_04 through rs_deny_30, excluding rs_deny_01_coverage and rs_deny_03_shadowing which have no `pub fn check`).

## Structure per rule

- **rule.rs** — production code (everything before `#[cfg(test)]`)
- **tests/helpers.rs** — test helper functions with `pub(super)` visibility, `use super::super::check` where needed, `CheckResult` imported explicitly
- **mod.rs** — `mod rule; pub use rule::{check}; #[cfg(test)] mod tests;`
- **tests/mod.rs** — prepended `mod helpers;`
- **Test files** — `super::super::` replaced with `super::helpers::` for helper items; `super::super::check` and `crate::inputs::ConfigDenyInput` used for non-helper references

## Decisions

- `pub(crate)` changed to `pub(super)` in helpers (narrower visibility, correct for test-only module)
- `#[cfg(test)]` attributes stripped from helpers content since the file is already inside a `#[cfg(test)]` module
- `ConfigDenyInput` references in test files changed to `crate::inputs::ConfigDenyInput` (was available via parent scope in original unsplit mod.rs)

## Key files

- All rule directories under `apps/guardrail3/crates/app/rs/families/deny/crates/runtime/src/{advisories,bans,licenses,sources}/rs_deny_*`

## Verification

All 146 tests pass: `cargo test -p guardrail3-app-rs-family-deny --lib`
