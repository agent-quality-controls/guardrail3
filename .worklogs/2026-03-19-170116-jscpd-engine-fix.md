# Fix non-walk-up coverage engine logic

**Date:** 2026-03-19 17:01
**Scope:** `src/commands/coverage/engine.rs`

## Summary
Fixed non-walk-up tools (jscpd) showing 54/55 dirs uncovered. A config at root now correctly covers all source dirs beneath it.

## Context & Problem
The engine's non-walk-up branch checked `cf.parent() == Some(dir)` — only matching source dirs at the exact same level as the config file. For jscpd at root, only root-level TS files matched. All subdirs showed uncovered despite jscpd scanning everything from CWD.

## Decisions Made
- **Chose:** `dir.starts_with(config_dir)` — config covers all descendant dirs
- **Why:** Non-walk-up tools scan everything under CWD. Config at root means everything is covered.
- **Alternatives considered:**
  - Add a `covers_all` trait method — over-engineering, starts_with is the correct general behavior for all non-walk-up tools
