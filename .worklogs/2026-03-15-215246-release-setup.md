# Release setup for guardrail3

**Date:** 2026-03-15 21:52
**Scope:** Cargo.toml, release-plz.toml, GitHub workflows, README, .gitignore

## Summary

Set up guardrail3 for crates.io publishing and binary distribution. Added publish metadata, GitHub Actions workflows (release-plz, binary cross-compile, CI), README, binstall metadata.

## Context & Problem

guardrail3 had `publish = false` and no release infrastructure. Needed full setup to publish to crates.io and distribute binaries.

## Decisions Made

### Dual license
- **Chose:** MIT OR Apache-2.0 (already in Cargo.toml, LICENSE files added in previous commit)

### Binary targets
- **Chose:** 4 targets: linux x86_64/aarch64, macOS x86_64/aarch64
- **Why:** Covers most developer machines. Windows excluded for now (cross-compilation complexity).

### Release automation
- **Chose:** release-plz with cargo-semver-checks
- **Why:** Standard Rust ecosystem tooling, catches breaking changes at code level

## Key Files for Context
- `Cargo.toml` — publish metadata + binstall
- `.github/workflows/release.yml` — release-plz automation
- `.github/workflows/binary-release.yml` — cross-compiled binaries
- `.github/workflows/ci.yml` — build + test + clippy + publish dry-run
- `release-plz.toml` — release configuration
- `README.md` — project documentation
