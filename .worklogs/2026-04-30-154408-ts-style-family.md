# TS style family

## Summary

Implemented a framework-independent `style` family for G3TS. The family checks that style enforcement is delegated to Stylelint and ESLint instead of parsing Tailwind classes inside guardrails.

## Decisions made

- Added `stylelint-config-parser` because Stylelint exposes `resolveConfig`; checking config text would have been a hand-rolled parser and would miss effective config behavior.
- Made `[ts.style]` the app/package contract with `source_globs`, `tailwind_denylist`, and `stylelint_css_globs`.
- Kept Tailwind misuse detection delegated to `eslint-plugin-tailwind-ban`; G3TS only verifies the effective ESLint rule wiring and denylist.
- Kept CSS rules delegated to Stylelint; G3TS only verifies the package stack, resolved config, strict a11y rule names, and `lint:css` fail-closed script.
- Wired `style` through the config runner instead of the structure runner to avoid expanding the already dense structure runner dependency surface.
