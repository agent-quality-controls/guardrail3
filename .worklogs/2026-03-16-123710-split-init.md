# Split init into rs/ts, remove monorepo profile, move ValidateDomains

**Date:** 2026-03-16 12:37
**Scope:** cli.rs, init.rs, main.rs, report/types.rs, validate modules, CLAUDE.md

## Summary
Complete separation of Rust and TypeScript stacks. `guardrail3 rs init` and `guardrail3 ts init` are now independent commands. Monorepo profile removed (use both init commands separately). ValidateDomains moved from rs/validate to shared report/types.

## Decisions Made
- `rs init` creates [rust] + [local] config and Rust override files. `ts init` creates/appends [typescript] section only.
- Monorepo profile was identical to service everywhere except init template — no longer needed.
- ValidateDomains is a check-category filter, not Rust-specific — belongs in shared types.
- Kept `guardrail3 init` as convenience (calls both rs init + ts init) for backward compat.
