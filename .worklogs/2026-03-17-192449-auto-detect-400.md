# Auto-detect TS app type + 400-line limit

**Date:** 2026-03-17 19:24
**Scope:** mod.rs, source_scan.rs, ts_source_scan_test.rs

## Summary
Added auto-detection of TypeScript app type from directory structure and package.json. Changed TS file length limit from 300 to 400.

## Auto-detection signals
1. `src/modules/domain/` exists → Service
2. `content/` directory exists → Content
3. Backend frameworks in deps (express/fastify/hono) → Service
4. Content tools in deps (velite/@mdx-js/contentlayer/nextra) → Content
5. `next` in deps without hex arch → Content
6. No signal → Service (safe fallback)

Config always wins over auto-detection.
