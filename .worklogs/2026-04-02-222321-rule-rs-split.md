# Split 203 rule mod.rs into mod.rs (facade) + rule.rs (implementation)

**Date:** 2026-04-02 22:23

## Summary
Split 203 rule mod.rs files. Production code compiles clean. Test compilation
has ~208 errors across 7 families — tests reference types/imports that were
in mod.rs but are now in rule.rs behind the facade.

## What was done
- Python script split production code to rule.rs, kept test helpers in mod.rs
- Fixed super:: → crate:: in rule.rs files (57 files)
- Fixed pub(crate) re-exports (3 files)
- Fixed dangling #[cfg(test)] attributes
- Fixed test-only type re-exports for hexarch

## Remaining test issue
Tests use `super::CheckResult`, `super::SomeInputType`, etc. which used to
be in mod.rs (accessible via super::). Now mod.rs is a facade — only
`check` (and a few others) are re-exported. Tests need to import these
types via `crate::` paths or mod.rs needs to re-export them.

## Fix approach options
1. Each mod.rs re-exports everything tests need (tedious, many types)
2. Tests use `crate::` imports instead of `super::` (correct but touches many test files)
3. Add `pub use rule::*;` to mod.rs (re-exports everything — defeats facade purpose)
