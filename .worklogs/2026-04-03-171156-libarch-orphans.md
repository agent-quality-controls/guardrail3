# Resolve libarch orphans

Libarch family was removed but several rules still gated on `profile_name == "library"`.

## Generalized (profile guard removed)
- **CARGO-01/02**: `unreachable_pub` lint now required for all crates, not just library profile
- **CODE-29**: Large trait surface check now applies to all crates
- **DEPS-08**: Dependency allowlist check now applies to all crates; renamed titles from "library allowlist" to "dependency allowlist"

## Removed (redundant)
- **CODE-26**: Lib glob reexport — fully redundant with RS-ARCH-02 which catches broad re-exports for all crates
- **CODE-27**: Facade only lib — fully redundant with RS-ARCH-02 which checks lib.rs facade-only for all crates

## Still undecided
- **CLIPPY-14**: Library global state bans — debatable whether to generalize
- **CARGO-15**: Library rust-version — should tie to publishable, not library
