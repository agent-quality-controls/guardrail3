## Summary
Added R58 source scan for direct std::fs usage (belt-and-suspenders for clippy disallowed_methods hole). Fixed filter_non_comment_lines string literal bug. Split source_scan.rs from 860 to 247 lines. Exempted test files from R38/R40. Exempted .mjs from T30.

## Decisions Made

### R58: Direct std::fs usage scan
- **Chose:** Source-level grep for `use std::fs` and `std::fs::` calls
- **Why:** clippy's disallowed_methods does NOT catch `use std::fs; fs::read_to_string()`. This is a clippy bug/limitation. guardrail3 needs its own check as belt-and-suspenders.
- **Exemptions:** fs.rs (centralized module), modules/ (string literals), type references (Permissions, Metadata, DirEntry)

### filter_non_comment_lines string literal fix
- **Chose:** strip_string_literals() before checking for /* and */
- **Why:** `"/*"` in string literals was treated as block comment start, causing massive undercounting
