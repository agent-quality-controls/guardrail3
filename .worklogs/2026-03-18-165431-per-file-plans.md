# Per-file config plans for all managed file types

**Date:** 2026-03-18 16:54
**Scope:** .plans/by_file/ (15 plan files)

## Summary

Created detailed per-file analysis plans for all 15 config file types guardrail3 manages. Each plan specifies: where the file lives, how the tool resolves it, what guardrail3 owns vs what the user owns, the merge/scaffold/validate algorithm, override mechanism, and edge cases. Each plan was adversarially reviewed and fixes applied.

## Decisions Made

### clippy.toml
- **Chose:** One per workspace root, merge-managed via toml_edit
- **Why:** Per-crate clippy.toml shadows workspace completely; Cargo.toml deps handle crate isolation
- Thresholds: warn if looser, don't force. Bans: enforce path presence, preserve user reasons.
- Overrides: removals only (user's extra bans preserved naturally by merge)
- Adversarial review found: garde conditionality unspecified, TOML table-array syntax, replacement/allow-invalid fields. All fixed.

### deny.toml
- **Chose:** One per workspace, merge-managed. [advisories] section 100% user-owned.
- **Why:** Advisory ignores are app-specific transitive dep issues
- Found: multiple-versions default wrong in code ("deny" should be "warn"), chrono ban too aggressive, "DO NOT EDIT" header contradicts merge model
- Adversarial review found: [bans.build] and [bans.workspace-dependencies] sections unaddressed. Fixed.

### eslint.config.mjs
- **Chose:** Shadow-imported (engine file + user config imports it)
- **Why:** Can't merge JS, can't overwrite 487 lines of project content
- Adversarial review found 5 bugs in current engine: missing files scoping on TS rules, missing stylisticTypeChecked, scaffold missing tsconfigRootDir, hex arch boundaries unscoped, content isolation not implemented. All documented.

### rustfmt.toml
- **Chose:** Merge-managed (ensure 7 keys, leave user extras)
- Adversarial review found: edition must be read from Cargo.toml on scaffold (not hardcoded "2024"). Fixed.

### Other files
- rust-toolchain.toml: merge-managed (ensure clippy+rustfmt in components)
- tsconfig.base.json: merge-managed (12 strict flags only)
- Per-app tsconfig.json: validate-only
- .stylelintrc.mjs: shadow-imported
- .npmrc: merge-managed (13 key=value)
- .jscpd.json: merge-managed (ignore patterns, not threshold)
- cspell.json: scaffold-once
- release-plz.toml, cliff.toml: scaffold-once (opt-in)
- pre-commit hook: scaffold-once
- package.json: validate-only
- Cargo.toml: validate-only
