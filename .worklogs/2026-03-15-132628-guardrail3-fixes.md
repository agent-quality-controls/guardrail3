## Summary
Fixed 4 issues found in the final completeness audit: rs validate scoping, ts generate stub, profile differentiation, and T60-T61 missing checks.

## Decisions Made

### rs validate scoping
- **Chose:** Call rs::validate::run() directly instead of commands::validate::run()
- **Why:** rs validate was running all stacks (Rust + TS + hooks). Should only run Rust checks, matching ts validate and hooks validate patterns.

### Profile differentiation
- **Chose:** Profile-specific method/type/ban selector functions in clippy.rs and deny.rs
- **Why:** library profile needs global-state bans everywhere + IO crate bans. minimal profile needs subset. monorepo same as service for Rust side.

### ts generate
- **Chose:** Generate .npmrc, tsconfig.base.json, .jscpd.json from canonical content, plus optional eslint starter
- **Why:** These are the TS-side canonical files that should be managed by guardrail3.
