# Add deny-toml parser crate

Historical note: this worklog predates the later parser-package rename.
The package is now `packages/deny-toml-parser/`, not `packages/deny-toml/`.

**Date:** 2026-04-03 23:18

## Summary
Most complex standalone parser crate. 5 nested sections (graph, advisories,
bans, licenses, sources) plus output. 8 untagged enum types for array entries
(advisory ignore, ban deny/allow/skip, ban features, license exceptions).
17 structs total, 21 tests, zero clippy warnings.

## Decisions
- **Small structs use private fields + accessors** (BanDenyDetail, BanSkipDetail,
  BanAllowDetail, BanFeatureEntry, AdvisoryIgnoreDetail, LicenseException,
  LicensesPrivateConfig) to avoid CODE-31. Large config section structs
  (DenyConfig, GraphConfig, AdvisoriesConfig, BansConfig, LicensesConfig,
  SourcesConfig, OutputConfig) use pub fields since Deserialize requires it.
- **`#[serde(rename = "crate")]`** on BanDenyDetail.crate_name because `crate`
  is a Rust keyword — the field name in TOML is `crate` but the Rust field
  is `crate_name`.
- **OutputConfig** added as optional section even though the plan didn't
  enumerate its fields — real deny.toml files can have `[output]`.
- **Added escape hatches** for all 6 parser crate packages' workspace
  Cargo.toml `redundant_pub_crate = "allow"` in guardrail3.toml. Previously
  only the app workspace had this escape hatch.

## Key files
- `packages/deny-toml-parser/crates/parser/types/src/` — parsed deny TOML types in the renamed package layout
- `packages/deny-toml-parser/crates/parser/runtime/src/` — parser runtime and tests in the renamed package layout
- `packages/deny-toml-parser/src/lib.rs` — public facade in the renamed package
- `guardrail3.toml` — added escape hatches for all parser package workspace Cargo.toml files
