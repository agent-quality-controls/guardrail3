# Node 24 GitHub Actions

## Summary

Updated Guardrail3 workflows to use Node 24-compatible GitHub checkout actions.
Fixed a fresh-CI compile failure in the release config semver rule.

## Decisions Made

- Changed all `actions/checkout@v4` references to `actions/checkout@v6`.
- Left Rust, release-plz, and release upload actions unchanged because the known warning concerns GitHub first-party JavaScript actions using Node 20.
- Used a clean worktree from `origin/main` so this CI maintenance change stays separate from the existing dirty development worktree.
- Re-enabled the GitHub `CI` workflow because it was disabled manually and could not report PR checks.
- Replaced root-level Cargo CI commands with `apps/guardrail3-rs/Cargo.toml` commands because the repository root has no `Cargo.toml`.
- Kept behavior fixture replay as a local verification gate instead of a CI step because those fixtures assert host tool inventory and are intentionally environment-sensitive.
- Replaced an unused `version` binding with `_` in the valid semver rule because fresh CI treats warnings as errors.
