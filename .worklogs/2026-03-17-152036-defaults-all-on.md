# All check categories on by default + mutants setup + warning fixes

**Date:** 2026-03-17 15:20
**Scope:** domain/report.rs, main.rs, commands/validate.rs, commands/init.rs, help_gen.rs, release_bin_checks.rs, release-plz.toml, .cargo/mutants.toml, Cargo.toml, adversarial_categories.rs

## Summary
Changed all check category defaults from off to on. Added cargo-mutants config. Fixed R-BIN-03 false positive on publish=false crates. Fixed release-plz.toml missing guardrail3 package.

## Decisions Made

### All categories on by default
- **Chose:** architecture=true, garde=true, tests=true, release=true
- **Why:** guardrail3 is opinionated — everything should be validated by default. Projects that don't need a category can disable it explicitly.

### R-BIN-03 skip publish=false crates
- **Chose:** Early return in check_binstall_metadata when publishable=false
- **Why:** Fuzz crates and internal tools with publish=false should not get release-readiness warnings about binstall metadata.

### R-TEST-08 left as warning
- **Chose:** Don't add mutation testing to pre-commit hook
- **Why:** cargo-mutants takes 10+ minutes to run. It should be run manually or in CI, not on every commit.
