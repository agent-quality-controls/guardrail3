## Summary
Removed remaining semantic assertions from hexarch runtime sidecars for rules 02–04 by converting them to shared assertions-module helpers, while keeping tests focused on fixture setup and helper invocations.

## Context & Problem
`cargo test` for `guardrail3-app-rs-family-hexarch --lib` and `RS-TEST-16` validation had failures and diagnostics still pointing at sidecars 02–04 after a prior partial cleanup. The requested scope was to move semantic checks for field/title/message/file/rule-id into owned assertions modules and keep sidecars as setup+helper calls only.

## Decisions Made

### Scope-constrained Remediation
- **Chose:** Update only the scoped rule 02–04 sidecar files to stop direct result inspection and fix helper-call compatibility issues.
- **Why:** User explicitly limited changes to 02–04 rule sidecars and matching assertions modules.
- **Alternatives considered:**
  - Reworking assertions modules and unrelated rules (rejected) to avoid touching outside scope.
  - Continuing with hand-rolled `assert!` checks in sidecars (rejected) because they are the source of RS-TEST-16.

### Minimal Fix Pattern
- **Chose:** Use existing assertion helpers (`assert_no_error`, `assert_error_count_matching[_file|prefix]`, `assert_error_summary`) and convert temporary `String` values used in arrays to `&str` via local variables (`inner_* .as_str()`).
- **Why:** This keeps sidecars ownership-safe and avoids type errors or lifetime issues while retaining behavior.
- **Alternatives considered:**
  - Inlining helper calls with direct string formatting in every array (rejected) because of compile-time mismatched type errors.
  - Keeping manual count checks (`errors.len()`) where not needed (rejected) to reduce semantic assertions in sidecars.

### Compile Hygiene
- **Chose:** Remove unused local `errors` variables and extra `assert_error_summary` arguments to align with current helper signatures.
- **Why:** Prevent warnings as errors and helper arity mismatches from blocking validation/test command execution.
- **Alternatives considered:**
  - Suppressing warnings (rejected) because build enforces `-D warnings` in this workspace.

## Architectural Notes
- The changes retain the test-module boundary model: sidecars now create fixtures/mutations and delegate semantics to rule-owned assertions modules.
- Existing assertions modules for rules 02–04 remain the semantic contract points; sidecars remain thin orchestration-style tests.
