# Fix T-TEST-04 and T-TEST-05 always parsing as TypeScript, not TSX

**Date:** 2026-03-16 10:29
**Task:** Bug fix — TSX files with JSX syntax may fail tree-sitter parse and fall through to grep

## Goal
Both `check_skip_without_reason_content` and `check_only_in_source_content` should use `parse_ts_file(content, is_tsx)` instead of `parse_typescript(content)`, so TSX files parse correctly via tree-sitter.

## Approach
1. Both functions already receive `filename: &str` — extract extension from that to determine `is_tsx`
2. Replace `ast_helpers::parse_typescript(content)` with `ast_helpers::parse_ts_file(content, is_tsx)` where `is_tsx = filename.ends_with(".tsx")`
3. Add test with TSX content containing JSX that would fail TS parse
