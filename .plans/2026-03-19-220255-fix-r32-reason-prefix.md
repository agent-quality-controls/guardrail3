# Fix HIGH-R32: Require `// reason:` prefix for #[allow] comments

**Date:** 2026-03-19 22:02
**Task:** R32-R33 comment detection accepts any `//` comment. Must require `// reason:` prefix.

## Goal
`#[allow(...)]` lines should only pass (R33 Info) if the comment starts with `// reason:`. Any other comment (`// lol`, `// this is fine`) should fail as R32 Error.

## Approach

### Step-by-step plan
1. In `check_item_level_allow_ast`, change the `has_comment` check from `l.contains("//")` to a case-insensitive check for `// reason:` after the `//`.
2. Update the reason extraction to pull text after `// reason:` instead of after `//`.
3. Verify existing tests pass (the R33 test already uses `// reason: test`).

### Key decisions
- **Case-insensitive match:** Use `.to_ascii_lowercase()` on the comment portion to match `// reason:`, `// Reason:`, `// REASON:` etc.
- **No regex:** Project bans regex crate. Use string methods.
