# Add property-based testing with proptest

**Date:** 2026-03-15 21:29
**Task:** Add property-based tests to guardrail3

## Goal
Property-based tests that generate random inputs and verify invariants: no panics, consistent results, correct detection of patterns.

## Approach

### Key constraint
This is a binary-only crate (no lib.rs). All modules are private. To write integration tests that access internal types, I need to add a `lib.rs` that re-exports the needed modules.

### Step-by-step plan
1. Add `proptest = "1"` to dev-dependencies in Cargo.toml
2. Create `src/lib.rs` that re-exports modules needed by property tests (config, report, rs::validate, discover, fs)
3. Update `main.rs` to use `guardrail3::` paths for the re-exported modules
4. Create `tests/property_tests.rs` with all 11 property tests

### Key decisions
- **Add lib.rs for testability:** Binary crate modules are inaccessible from integration tests. Adding lib.rs that re-exports public modules is the standard Rust pattern for testing binary crates.
  - Alternative: Put all tests as unit tests inside existing files -- rejected because it would bloat individual files and mix concerns.
  - Alternative: Test only via CLI binary -- rejected because many properties need access to internal types (CheckResult, Severity, etc.)

## Files to Modify
- `Cargo.toml` -- add proptest dev-dependency
- `src/lib.rs` -- new file, re-export modules for testing
- `src/main.rs` -- update to use lib re-exports
- `tests/property_tests.rs` -- new file, all property tests
