# Fix R58 glob bypass and R30 inline module inner attrs

**Date:** 2026-03-19 22:04
**Task:** Fix two HIGH issues in Rust source scan

## Goal
1. R58: `use std::*` (glob import) should be detected as importing `std::fs`
2. R30: `#![allow(...)]` inside inline `mod foo { ... }` blocks should be detected

## Approach

### Fix 1: R58 glob — ast_helpers.rs
In `use_subtree_is_fs`, change `UseTree::Glob(_) => false` to `UseTree::Glob(_) => true`.
Rationale: this function is only called after `use_tree_matches_std_fs` has already confirmed we're inside `use std::`. A glob at any level within `std::` means `fs` is being imported (e.g., `use std::*` imports everything including `fs`).

### Fix 2: R30 inline mod — ast_helpers.rs + allow_checks.rs
Add a new function `find_inline_mod_allows` to ast_helpers that uses a syn visitor to find `#![allow(...)]` inner attributes on `ItemMod` items with bodies. Then call it from `check_crate_level_allow` in allow_checks.rs.
