## Conventions

Every check follows this pattern:
```rust
CheckResult {
    id: "CHECK-ID".to_owned(),
    severity: Severity::Xxx,
    title: "short title".to_owned(),
    message: "detailed message".to_owned(),
    file: Some("path".to_owned()) | None,
    line: Some(n) | None,
}
```

Check functions: `fn check_xxx(workspace_root: &Path, ...) -> Vec<CheckResult>`
Or push-style: `fn check_xxx(workspace_root: &Path, ..., results: &mut Vec<CheckResult>)`

Tests: `#[cfg(test)] mod tests` in same file. Per check: one negative test (violation → finding emitted) + one positive test (no violation → no finding with that ID).

## Verification Script Spec

The verification script (`scripts/verify-domains.sh`) checks every deliverable mechanically.

### Per-check verification (repeated for all 37 checks):
```bash
verify_check() {
    local check_id="$1"
    local file="$2"
    local test_prefix="$3"

    # 1. Check ID exists in source
    grep -q "\"$check_id\"" "$file" || fail "$check_id: ID not in source"

    # 2. Test exists (negative case)
    grep -q "${test_prefix}_.*emits\|${test_prefix}_.*missing\|${test_prefix}_.*no_" "$file" || fail "$check_id: negative test missing"

    # 3. Test exists (positive case) — skip for Info-only inventory checks
    grep -q "${test_prefix}_.*pass\|${test_prefix}_.*has_\|${test_prefix}_.*present" "$file" || fail "$check_id: positive test missing"
}
```
