# Fix F-04-03 and HIGH-R30/R37: dependency sections + cfg_attr(all()) bypass

**Date:** 2026-03-19 22:03
**Task:** Fix two HIGH issues in guardrail3

## Goal
1. R-ARCH-02 dependency flow check should inspect `[dev-dependencies]` and `[build-dependencies]` in addition to `[dependencies]`
2. `cfg_attr(all(), allow(...))` should be treated as unconditional (Error/R30/R32) instead of conditional (Info/R37)

## Approach

### Issue 1: F-04-03 — Dependency direction only checks `[dependencies]`

**File:** `apps/guardrail3/src/app/rs/validate/hex_arch_checks.rs`

In `check_dependency_flow()`, line 178 does:
```rust
let Some(deps) = table.get("dependencies").and_then(|d| d.as_table()) else { continue; };
```

Change to iterate over all three sections: `dependencies`, `dev-dependencies`, `build-dependencies`. The message should indicate which section the violation is in.

Also update `check_library_service_boundary()` (line 286) which has the same single-section limitation.

### Issue 2: HIGH-R30/R37 — `cfg_attr(all(), allow(...))` bypass

**Files:**
- `apps/guardrail3/src/app/rs/validate/ast_helpers.rs` — modify `extract_cfg_attr_allow_lints` to also return whether the condition is always-true
- `apps/guardrail3/src/app/rs/validate/allow_checks.rs` — in `check_cfg_attr_allow_ast`, use the new info to emit R30/R32 instead of R37

The approach: modify `find_cfg_attr_allows` to return a richer type that includes an `is_always_true` flag. The `extract_cfg_attr_allow_lints` function parses the cfg_attr tokens — we need to check if the first argument (before the comma) is `all()` with empty parens.

Alternative: Keep `Located` but add a new parallel function. Rejected — changing the type is cleaner.

**Decision:** Create a new type `CfgAttrAllow` with `(line, lint, is_always_true)` instead of modifying `Located`.
