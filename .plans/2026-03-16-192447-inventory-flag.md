# Add --inventory flag to hide non-actionable confirmations

**Date:** 2026-03-16 19:24
**Task:** Add `--inventory` flag that controls visibility of passing/confirmation results

## Goal
When running `guardrail3 validate` without `--inventory`, passing/confirmation Info results are hidden — only problems, warnings, errors, and audit-trail items (suppressions, cop-outs) are shown. With `--inventory`, everything is shown (current behavior).

## Approach

### Step 1: Add `inventory` field to CheckResult
In `domain/report.rs`, add `pub inventory: bool` to `CheckResult`. Default false. Add `as_inventory()` builder method.

### Step 2: Bulk-add `inventory: false` to all 336 CheckResult constructions
Every `CheckResult {` literal needs `inventory: false`. Use sed/regex replacement across all 38 files.

### Step 3: Add `--inventory` CLI flag
In `cli.rs`, add `pub inventory: bool` to `ValidateArgs`.

### Step 4: Thread `show_inventory` bool through reporters
Change all three `print_report` functions to accept `show_inventory: bool`. Filter results where `inventory == true` when `show_inventory` is false.

### Step 5: Update `print_report` dispatch in `main.rs` and `commands/validate.rs`
Pass `args.inventory` to reporters.

### Step 6: Update summary to show inventory count
Show "X info (Y hidden, use --inventory to show all)" when items are hidden.

### Step 7: Mark inventory results
Go through check modules and add `.as_inventory()` to Info-severity confirmation results. The rule: `inventory = true` ONLY when severity == Info AND it's a "check passed" confirmation. NEVER for suppressions (R33, R35, R36), cop-outs, or Error/Warn.

### Step 8: Add `inventory` to JSON output
Include the field in serde JSON serialization.

## Key Decisions
- **Builder method over Default:** Can't use Default for CheckResult (no sensible defaults for id/title/etc). Builder method `.as_inventory()` is chainable and explicit.
- **Filter in reporter, not in report:** Report always contains all results for counting. Reporters filter display based on flag.
