# Forms - MVP

## Decision

- Use React Hook Form + Zod
- Validation schemas live in `types/`
- Simple forms use server actions
- Long-running or client-lifecycle-heavy flows use TanStack Query mutations

## Why

- React Hook Form + Zod is the standard low-friction setup for this stack
- It works well with Next.js App Router
- It avoids overdesigning form infrastructure for MVP
- It supports both simple synchronous-looking submits and long-running workflow triggers

## Schema placement

- Form input schemas live in `types/`
- The same schema is used for:
  - client-side validation
  - server-side validation
  - typed input parsing at boundaries

## Component split

- Presentational form components stay dumb
- Form orchestration lives outside the presentational layer
- `components/` renders fields and form UI
- `app/` or a form-specific adapter/action layer owns submission behavior

## Server actions

Use server actions for:
- small CRUD
- settings updates
- forms that submit, validate, save, and return a bounded result

Do not default to server actions for:
- long-running workflows
- flows that need explicit client-side lifecycle handling
- flows that immediately transition into polling or multi-stage async UI

## TanStack Query mutations

Use client mutations for:
- workflow-triggering forms
- long-running actions
- flows that need retries, status control, or client-managed follow-up behavior
- actions that naturally lead into polling

For Websmasher MVP, crawl submission should default to this path.

## Validation and errors

- Zod is the source of truth for input validation
- Server always re-validates input
- Validation failures return structured field errors
- Expected validation failures are not thrown as unexpected exceptions

## What not to do

- Do not let each form invent its own validation shape
- Do not scatter validation logic into presentational components
- Do not build a generic form engine for MVP
- Do not force all forms through server actions just because the framework supports them

## Summary

- React Hook Form + Zod
- Schemas in `types/`
- Server actions for simple forms
- TanStack Query mutations for workflow-style forms
