# Standalone config parser crates

## Goal

Create standalone, reusable parser crates for Rust tool configuration files
that have no standard parser. Each crate follows the `cargo_toml` crate's
design patterns and is independently publishable.

## Design patterns (from cargo_toml)

All parser crates follow these conventions:

```rust
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(rename_all = "kebab-case")]
pub struct Config {
    pub field: Option<String>,
    #[serde(default)]
    pub list: Vec<Item>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,  // catch-all, no deny_unknown_fields
}

impl Config {
    pub fn from_str(content: &str) -> Result<Self, Error> { ... }
    pub fn from_path(path: impl AsRef<Path>) -> Result<Self, Error> { ... }
}
```

Rules:
- `#[serde(rename_all = "kebab-case")]` on all structs (TOML is kebab-case)
- `#[serde(default)]` for optional collections (empty Vec, not Option<Vec>)
- `#[serde(flatten)] extra: BTreeMap<String, Value>` — forward compat, no `deny_unknown_fields`
- `#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]` on all types
- `from_str()` / `from_path()` constructors
- Custom `Error` enum wrapping `toml::de::Error`
- Re-export `toml::Value`
- Use `BTreeMap` for deterministic ordering
- `#[non_exhaustive]` on main structs

## 1. clippy-toml

**Crate:** `packages/clippy-toml-parser/`
**Schema complexity:** Flat key-value + arrays of ban entries

### Types

```rust
pub struct ClippyConfig {
    // Thresholds
    pub max_struct_bools: Option<u32>,
    pub max_fn_params_bools: Option<u32>,
    pub too_many_lines_threshold: Option<u32>,
    pub too_many_arguments_threshold: Option<u32>,
    pub excessive_nesting_threshold: Option<u32>,
    pub cognitive_complexity_threshold: Option<u32>,
    pub type_complexity_threshold: Option<u32>,

    // Ban lists
    #[serde(default)]
    pub disallowed_methods: Vec<BanEntry>,
    #[serde(default)]
    pub disallowed_types: Vec<BanEntry>,
    #[serde(default)]
    pub disallowed_macros: Vec<BanEntry>,

    // Test relaxations
    pub allow_dbg_in_tests: Option<bool>,
    pub allow_print_in_tests: Option<bool>,
    pub allow_expect_in_tests: Option<bool>,
    pub allow_panic_in_tests: Option<bool>,
    pub allow_unwrap_in_tests: Option<bool>,

    // Library settings
    pub avoid_breaking_exported_api: Option<bool>,

    // Other known keys
    pub msrv: Option<String>,

    // Catch-all
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

/// Ban entry: either a plain path string or {path, reason} table.
#[serde(untagged)]
pub enum BanEntry {
    Simple(String),
    Detailed(BanEntryDetail),
}

pub struct BanEntryDetail {
    pub path: String,
    pub reason: Option<String>,
}
```

### Guardrail3 uses
- RS-CLIPPY-02..22: threshold validation
- RS-CLIPPY-04/05/06/07/20: ban list presence/extras
- RS-CLIPPY-08/15: ban entry reason quality
- RS-CLIPPY-17: test relaxation settings
- RS-CLIPPY-16: avoid-breaking-exported-api
- RS-CLIPPY-19: unknown key detection (via `extra` field)
- RS-CLIPPY-25: parse success

## 2. deny-toml

**Crate:** `packages/deny-toml-parser/`
**Schema complexity:** Nested tables with arrays of complex entries

### Types

```rust
pub struct DenyConfig {
    pub graph: Option<GraphConfig>,
    pub advisories: Option<AdvisoriesConfig>,
    pub bans: Option<BansConfig>,
    pub licenses: Option<LicensesConfig>,
    pub sources: Option<SourcesConfig>,
    pub output: Option<OutputConfig>,

    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

pub struct GraphConfig {
    pub all_features: Option<bool>,
    pub no_default_features: Option<bool>,
    #[serde(default)]
    pub targets: Vec<Value>,
    #[serde(default)]
    pub exclude: Vec<String>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

pub struct AdvisoriesConfig {
    pub unmaintained: Option<String>,   // deny/warn/allow
    pub yanked: Option<String>,
    pub vulnerability: Option<String>,  // deprecated
    pub notice: Option<String>,         // deprecated
    pub unsound: Option<String>,        // deprecated
    #[serde(default)]
    pub ignore: Vec<AdvisoryIgnoreEntry>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

pub struct BansConfig {
    pub multiple_versions: Option<String>,
    pub wildcards: Option<String>,
    pub allow_wildcard_paths: Option<bool>,
    pub highlight: Option<String>,
    #[serde(default)]
    pub deny: Vec<BanDenyEntry>,
    #[serde(default)]
    pub allow: Vec<BanAllowEntry>,
    #[serde(default)]
    pub skip: Vec<BanSkipEntry>,
    #[serde(default)]
    pub features: Vec<BanFeatureEntry>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

pub struct LicensesConfig {
    #[serde(default)]
    pub allow: Vec<String>,
    pub confidence_threshold: Option<f64>,
    #[serde(default)]
    pub exceptions: Vec<LicenseException>,
    pub private: Option<LicensesPrivateConfig>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

pub struct SourcesConfig {
    pub unknown_registry: Option<String>,
    pub unknown_git: Option<String>,
    #[serde(default)]
    pub allow_registry: Vec<String>,
    #[serde(default)]
    pub allow_git: Vec<String>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

// Entry types for arrays
#[serde(untagged)]
pub enum BanDenyEntry {
    Simple(String),
    Detailed(BanDenyDetail),
}
pub struct BanDenyDetail {
    pub name: Option<String>,
    #[serde(rename = "crate")]
    pub crate_name: Option<String>,
    pub version: Option<String>,
    #[serde(default)]
    pub wrappers: Vec<String>,
    pub reason: Option<String>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}
// ... similar for BanSkipEntry, LicenseException, etc.
```

### Guardrail3 uses
- RS-DENY-04: deprecated advisory fields
- RS-DENY-05/06: advisory baseline/stricter
- RS-DENY-07/08: graph settings
- RS-DENY-09/10/11/12/13: bans settings
- RS-DENY-14..17: license settings
- RS-DENY-18..20: sources settings
- RS-DENY-23/24: skip/ignore entry structure
- RS-DENY-25: allow overrides
- RS-DENY-28: unknown keys (via `extra` fields at every level)
- RS-DENY-29: ignore accumulation
- RS-DENY-30: managed wrappers

## 3. rustfmt-toml

**Schema complexity:** Flat key-value (simplest)

### Types

```rust

    // Core managed keys

    pub reorder_imports: Option<bool>,
    pub reorder_modules: Option<bool>,
```

- RS-FMT-02: required settings validation

- RS-FMT-04: nightly-only keys on stable
- RS-FMT-05: nested config files
- RS-FMT-07: ignore escape hatch validation

## 4. nextest-toml

**Crate:** `packages/nextest-toml-parser/`
**Schema complexity:** Complex but guardrail3 uses very little

### Types

```rust
pub struct NextestConfig {
    #[serde(default)]
    pub profile: BTreeMap<String, NextestProfile>,

    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

pub struct NextestProfile {
    pub slow_timeout: Option<TimeoutConfig>,
    pub leak_timeout: Option<TimeoutConfig>,
    pub test_threads: Option<Value>,
    pub retries: Option<Value>,
    pub fail_fast: Option<bool>,

    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}

#[serde(untagged)]
pub enum TimeoutConfig {
    Simple(String),          // "60s"
    Detailed(TimeoutDetail), // { period = "60s", terminate-after = 2 }
}

pub struct TimeoutDetail {
    pub period: String,
    pub terminate_after: Option<u32>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, Value>,
}
```

## Migration order

By complexity (simplest first):
1. **rustfmt-toml** — flat key-value, simplest schema, 8 managed keys
2. **mutants-toml** — flat key-value, 10 keys
3. **clippy-toml** — flat + ban entry arrays (untagged enum)
4. **nextest-toml** — nested profiles, timeout config variants
5. **deny-toml** — most complex: 5 nested sections, arrays of complex entries

## Implementation per crate

Each crate:
1. `Cargo.toml` with `serde`, `toml` dependencies
2. `src/lib.rs` with types + `from_str()`/`from_path()`
3. `src/error.rs` with custom Error wrapping toml::de::Error
4. `tests/` with round-trip and real-world config samples
5. No dependency on guardrail3 — fully standalone
