**Date:** 2026-03-16 08:31
**Task:** Rewrite check_direct_fs_usage to use syn-based AST parsing via ast_helpers

## Goal
R58 check uses syn to detect `use std::fs` imports, eliminating false positives on comments and string literals. Falls back to grep on parse failure.

## Approach

1. Add `use super::ast_helpers;` import to code_quality_checks.rs
2. Rewrite `check_direct_fs_usage` body:

   - Try `ast_helpers::parse_file(content)`
   - If Some: use `find_std_fs_imports` for use-imports, keep grep for inline `std::fs::` calls (these aren't use-items so syn won't find them) but only outside #[cfg(test)]
   - If None: fall back to existing grep logic
3. The inline `std::fs::` call detection (second half) stays as grep since those are expressions, not use-items
