# Full Hex Arch — Ports, Traits, Proper Boundaries

**Date:** 2026-03-16 13:43
**Task:** Complete hex arch with trait-based ports following steady-parent pattern

## Target Structure
```
src/
  domain/                    — pure types, zero I/O
    report.rs               — CheckResult, Section, Report, Severity, ValidateDomains
    config/                 — guardrail3.toml types
    modules/                — embedded config content (const strings)

  ports/
    mod.rs
    outbound.rs             — trait FileSystem { read_file, list_dir, metadata, read_file_err }
                            — trait ToolChecker { is_installed, run_cargo_metadata }

  app/                      — business logic, uses ports via &dyn traits
    validate/
      rs/                   — Rust validation (takes &dyn FileSystem)
      ts/                   — TypeScript validation (takes &dyn FileSystem)
      hooks/                — Hook validation (takes &dyn FileSystem + &dyn ToolChecker)
    discover.rs             — stack detection (takes &dyn FileSystem)

  adapters/
    inbound/
      cli/                  — clap CLI (main.rs, cli.rs, help_gen.rs)
      cli/commands/         — command handlers
    outbound/
      fs.rs                 — FileSystem impl using std::fs + walkdir
      tool_runner.rs        — ToolChecker impl using std::process::Command
      report/               — text, json, markdown formatters

  lib.rs                    — public API, wires ports to adapters
  main.rs                   — CLI entry point
```

## Outbound Port: FileSystem trait
```rust
pub trait FileSystem {
    fn read_file(&self, path: &Path) -> Option<String>;
    fn read_file_err(&self, path: &Path) -> Result<String, std::io::Error>;
    fn list_dir(&self, path: &Path) -> Vec<DirEntry>;
    fn metadata(&self, path: &Path) -> Option<std::fs::Metadata>;
    fn walk_files(&self, path: &Path, extensions: &[&str]) -> Vec<PathBuf>;
}
```

## Outbound Port: ToolChecker trait
```rust
pub trait ToolChecker {
    fn is_installed(&self, tool: &str) -> bool;
    fn cargo_metadata(&self, path: &Path) -> Option<String>;
}
```

## Migration Steps
1. Define ports (traits) in src/ports/
2. Move domain types to src/domain/ (already done in task 54)
3. Move validation logic to src/app/ — update all functions to accept &dyn FileSystem
4. Move CLI + formatters to src/adapters/
5. Implement FileSystem and ToolChecker in adapters/outbound/
6. Wire everything in lib.rs and main.rs
7. Add architectural boundary tests
8. Add comprehensive CLI help (task 56)

## What changes in validation functions
Every function that calls crate::fs::read_file(path) changes to fs.read_file(path)
where fs: &dyn FileSystem is passed as a parameter.

The orchestrator functions (rs::validate::run, ts::validate::run) already take path: &Path.
They'll additionally take fs: &dyn FileSystem.
