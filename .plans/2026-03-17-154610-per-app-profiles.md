# Per-app type profiles for TypeScript

**Date:** 2026-03-17 15:46
**Task:** Add per-app type configuration so different apps in a monorepo get different checks

## Goal
```toml
[typescript.apps.landing]
type = "content"              # enables content/accessibility/SEO checks, skips T-ARCH/T50

[typescript.apps.admin]
type = "service"              # enables hex arch, route wrappers

[typescript.apps.shared-ui]
type = "library"              # dependency restrictions, no app-specific checks
```
Profiles determine which checks fire per-app. Override escape hatch: `checks.architecture = true` on a service that also needs it.

## Approach

### 1. Config schema (domain/config/types.rs)
- Add `TsAppConfig { type_: Option<String>, checks: Option<TsChecksConfig> }`
- Change `TypeScriptConfig.apps` from `Option<Vec<String>>` to `Option<BTreeMap<String, TsAppConfig>>`
- Keep backward compat: if apps is not configured, auto-discover

### 2. App type enum (new: domain/ts_app_type.rs or in report.rs)
```rust
pub enum TsAppType {
    Service,    // hex arch, route wrappers, full ESLint strict
    Content,    // content checks, SEO, accessibility, static gen
    Library,    // dep restrictions, no app-specific checks
}
```
With a method: `fn default_categories(&self) -> TsCheckCategories` that returns what each type enables.

### 3. Resolved per-app context
```rust
pub struct TsAppContext {
    pub name: String,
    pub path: PathBuf,
    pub app_type: TsAppType,
    pub categories: TsCheckCategories,  // merged: type defaults + per-app overrides + global
}
```

### 4. TS validate mod.rs changes
- Discover apps (existing logic)
- For each app, resolve its type from config (or auto-detect)
- Build per-app categories: type defaults → override with per-app checks → override with CLI
- T-ARCH checks: only run on Service apps
- Future T-CONTENT checks: only run on Content apps
- Global checks (config_files, source_scan, test_checks): run on all

### 5. What changes NOW vs LATER
**Now:** Config schema, type enum, per-app category resolution, T-ARCH gated on type=service
**Later:** T-CONTENT-01 through T-CONTENT-19, accessibility checks, auto-detection
