# Plan: Fix test assertions after rule message/severity audit

## Summary

The rule message audit changed error message strings, severity levels, and inventory flags across multiple families. Tests that assert on exact message content or severity values now fail. Total: ~85 failing tests across 6 families.

## Failure categories

### Category 1: Message string changes (most common)
Tests assert `contains("old message substring")` or exact message match. The rule message was appended with a fix-action sentence.

**Approach:** For each failing test, read the test assertion, find the old expected string, update it to match the new message. Most of these are mechanical — the old substring is still present, just with an appended sentence. Some tests check exact full message and need the full new string.

**Affected rules:**
- CODE-01, 05, 09, 10, 15, 23, 34, 35 (appended fix actions)
- CODE-31 (changed message to include threshold info)
- DEPS-05, 06, 07, 08, 09, 10, 12 (appended fix actions, changed "Dependency" to "Build/Dev dependency")
- CLIPPY-02, 03, 04, 05, 09, 10, 11, 13, 14, 15, 18, 19, 20, 21, 22, 24 (appended fix actions)
- CARGO-01, 14 (changed message wording)
- RELEASE PUB-04, 05, 06, 07, 13, 14, BIN-03 (appended fix actions)

### Category 2: Severity changes
Tests assert `Severity::Warn` but rule now emits `Severity::Error` (or vice versa).

**Approach:** Update the expected severity in test assertions.

**Affected rules:**
- CODE-07: Info → Warn (escape hatch visibility)
- CODE-08: Info → Warn (escape hatch visibility)
- CODE-13: unreachable! Info → Warn
- CODE-19: Info → Warn (large type inventory)
- CODE-23: Info → Warn (build-script include inventory)
- CODE-31: Always Error → Warn<5 / Error>=5
- HEXARCH-16: documented escape-hatch Error → Warn
- RELEASE-11: Warn → Error (accidentally publishable)
- PUB-04: Warn → Error (README missing)
- PUB-05: Warn → Error (README stub/no heading)
- PUB-06: Warn → Error (keywords)
- PUB-07: Warn → Error (categories)
- PUB-13: Info → Warn (docs.rs metadata)
- PUB-14: Info → Warn (include/exclude)

### Category 3: Inventory flag changes
Tests assert `.inventory == true` but rule now emits without `.as_inventory()`.

**Approach:** Same rules as severity changes — update assertions to expect `inventory: false`.

**Affected rules:** CODE-07, 08, 19, 23 (removed .as_inventory())

### Category 4: Logic changes (removed library profile guard)
Tests that set up library profile scenarios and expect the rule to fire. Now the rule fires for ALL profiles, so tests asserting "should not fire for service profile" will flip.

**Approach:** 
- CARGO-01: Test `local_library_profile_requires_unreachable_pub` needs renaming/updating — now fires for all profiles
- DEPS-08: Tests that check "warns only for library crates" need updating — now warns for all crates
- CLIPPY-14: Still gated on library (unchanged), but test construction may reference removed field

### Category 5: Removed rules (CODE-26, CODE-27)
Tests expect these rules to emit results, but the rules are now no-ops.

**Approach:** Update tests to expect zero results from these rules.

## Execution strategy

Work family by family. For each family:
1. Run `cargo test -p <package>` to see failures
2. Read each failing test to understand what it asserts
3. Update the assertion to match the new message/severity/inventory
4. Run the test again to confirm it passes
5. Move to next family

**Order:** clippy (27 failures, mostly mechanical message changes) → code (39 failures, mix of all categories) → deps (20 failures) → release (19 failures) → cargo (5 failures)

## Risk

- Some tests may be "attack tests" that verify the rule catches specific bypass patterns. Changing their assertions is correct as long as the rule logic didn't change — only the message/severity changed.
- Tests for CODE-26/27 that are now no-ops should probably be removed or converted to "no results" assertions, not just silenced.
- DEPS-08 tests that checked "only fires for library" now need to verify "fires for everything" — this changes the test intent, not just the assertion.
