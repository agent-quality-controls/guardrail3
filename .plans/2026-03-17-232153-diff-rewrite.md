# Rewrite diff.rs dry-run to show meaningful file status

**Date:** 2026-03-17 23:21
**Task:** Replace the current diff.rs "describe config" approach with smart file-level status display

## Goal
The `diff` command should show per-file status: would create, would update (with line diff count), has custom entries (for TOML files), or no changes. Custom entries in clippy/deny TOML files should be detected by comparing `{ path = ... }` / `{ name = ... }` lines.

## Approach

### Step-by-step plan
1. Remove old `has_changes`, `describe_rs_config`, `describe_ts_config`, `count_section_entries`, `describe_ban_categories`, `describe_hook_steps` functions
2. Add `show_smart_diff` function that iterates expected files, reads actual from disk, classifies each
3. Add `FileStatus` enum: Create, Update(usize), NoChanges, UpdateWithCustom { diff_lines, custom_entries }
4. Add `extract_custom_entries` for TOML files — compare `{ path = ...}` and `{ name = ...}` lines between expected and actual
5. Add `count_diff_lines` for generic files
6. Rewrite `run()` and `run_ts()` to call `show_smart_diff`
7. Every `#[allow]` gets `// reason:` on same line

### Key decisions
- **Single `show_smart_diff` function for both rs and ts**: Both `run` and `run_ts` use the same logic — just different `generate_expected` calls
- **Custom entry detection**: Parse lines matching `{ path = ` or `{ name = ` from both expected and actual, set-difference to find customs
- **Line diff count**: Simple line-by-line comparison count, not a full unified diff
