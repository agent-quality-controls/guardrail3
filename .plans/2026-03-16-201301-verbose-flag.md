# Add --verbose flag to summarize audit trail items

**Date:** 2026-03-16 20:13
**Task:** Add --verbose flag; when false (default), summarize high-count audit trail check IDs instead of listing each individually.

## Goal
Reduce text report from 56KB to compact output by summarizing check IDs with >3 results (R33, R35, R36, R19, R20) into single-line counts, with "(use --verbose to list each)" hint.

## Approach

### Step-by-step plan
1. **cli.rs** — Add `verbose: bool` flag to `ValidateArgs`
2. **report/text.rs** — Change `print_report` signature to accept `verbose: bool`, add summarization logic grouping by check ID, threshold >3
3. **report/markdown.rs** — Same summarization as text
4. **report/json.rs** — No change (always include all items per spec)
5. **main.rs** — Thread `args.verbose` through `print_report` calls
6. **help_gen.rs** — Add `--verbose` to command reference
7. **guide.rs** — Add `--verbose` to guide

### Key decisions
- **Threshold of 3**: If a check ID has <=3 results, show them. If >3, summarize.
- **JSON always full**: Machine-readable format should not lose data.
- **Markdown same as text**: Both human-readable formats get summarization.
- **Grouping by check ID**: The summarization groups results by `id` field, counts them, shows a single summary line.
