# Delete G3TS Dead Tests And Assertions

## Goal

Remove the remaining G3TS test-only code now that `g3ts-rule`, `g3ts-validate-repo`, and `g3ts-cli-output` fixtures cover the external CLI behavior surface.

## Scope

In scope:

- `apps/guardrail3-ts`
- `packages/ts`

Out of scope:

- `packages/rs`
- `packages/shared`
- `packages/parsers`
- `legacy`
- `behavior/fixtures`
- `behavior/golden`
- fixture replay and verifier scripts

## Current State

- `python3 scripts/behavior/list-rust-tests.py --format json` reports `0` active test functions.
- G3TS still has test-only directories and assertion crates:
  - `crates/assertions`
  - `*_tests`
  - `tests`
  - `contract_tests`
  - `#[cfg(test)]` module wiring

## Approach

1. Delete G3TS test-only directories.
   - Remove every in-scope `crates/assertions` directory.
   - Remove every in-scope source test sidecar directory named `*_tests`, `tests`, or `contract_tests`.

2. Remove G3TS test wiring.
   - Remove `#[cfg(test)]` module blocks from in-scope Rust source files.
   - Remove `#[path = "..._tests/mod.rs"] mod ...;` lines paired with those blocks.
   - Do not remove non-test production modules.

3. Remove Cargo references to deleted assertion crates.
   - Remove `"crates/assertions"` from workspace members.
   - Remove local assertion crate dependencies from `[dev-dependencies]` or dependency tables.
   - Keep runtime and types crates intact.

4. Verify no G3TS test-only surface remains.
   - No in-scope `crates/assertions` directories.
   - No in-scope `*_tests`, `tests`, or `contract_tests` directories.
   - No in-scope Rust files contain `#[cfg(test)]`.
   - No in-scope `Cargo.toml` references assertion crates.

5. Verify behavior stayed stable.
   - `fixture3 check --suite g3ts-rule`
   - `fixture3 check --suite g3ts-validate-repo`
   - `fixture3 check --suite g3ts-cli-output`
   - `python3 scripts/behavior/verify-g3ts-family-rule-fixtures.py`
   - `python3 scripts/behavior/verify-g3ts-rule-fixture-coverage.py`
   - `scripts/behavior/verify-all.sh`

6. Report G3TS fixture line counts.
   - Count tracked files and physical lines under `behavior/fixtures/g3ts-rule`.
   - Count tracked files and physical lines under `behavior/golden/g3ts-rule`.
   - Do not count `.fixture3`, reducer scratch, or generated received output.

## Done Criteria

- G3TS test-only files are deleted.
- Cargo manifests no longer reference deleted G3TS assertion crates.
- G3TS fixture verification passes.
- Full behavior verifier passes.
- Worklog is committed with code changes.
