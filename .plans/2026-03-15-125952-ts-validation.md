**Date:** 2026-03-15 12:59

## Goal
Full TypeScript project validation covering config files (ESLint, tsconfig, .npmrc, package.json, .jscpd.json), source code scanning (.ts/.tsx/.mjs files), and ESLint boundary audit.

## Approach

### Files to create/modify
1. `src/ts/validate/config_files.rs` — T1-T22, T50-T58 config file checks
2. `src/ts/validate/source_scan.rs` — T23-T35, T40-T49, T59 source scanning
3. `src/ts/validate/eslint_audit.rs` — T36-T39 boundary checks
4. `src/ts/validate/mod.rs` — orchestrator returning Report
5. `src/ts/mod.rs` — no change needed (already exports validate)
6. `src/commands/validate.rs` — wire TS validation into combined report
7. `src/main.rs` — pass ValidateArgs through to ts validate

### Pattern
Follow rs::validate pattern: each module returns Vec<CheckResult>, orchestrator collects into Sections in a Report.
