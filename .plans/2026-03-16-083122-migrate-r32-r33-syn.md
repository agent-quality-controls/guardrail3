# Migrate R32-R33 (item-level allow) from grep to syn

**Date:** 2026-03-16 08:31
**Task:** Rewrite check_item_level_allow to use syn AST parsing with grep fallback

## Goal
R32-R33 checks use syn's find_item_allows for accurate detection, falling back to grep only when parsing fails.

## Approach

### Step-by-step plan
1. Add `use super::ast_helpers` import to allow_checks.rs
2. Rewrite `check_item_level_allow` body:
   - Try `ast_helpers::parse_file(content)`
   - If Some: use `ast_helpers::find_item_allows(&file)` to get `(line, lint)` pairs
   - For each: index into source lines at that line to check for `//` comment
   - Emit R33 (Info) if comment found, R32 (Error) if not
   - If None: fall back to existing grep logic (extracted to helper)
3. Extract grep logic to `check_item_level_allow_grep` private fn

### Key decisions
- **Line indexing:** syn returns 1-based lines; source lines vec is 0-based, so subtract 1
- **Comment check:** Look at the raw source line (not filtered), since `//` on attribute line is what matters
- **Fallback preserves behavior:** grep path is identical to current code
