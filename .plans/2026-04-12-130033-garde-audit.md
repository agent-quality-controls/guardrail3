# Garde Audit

## Goal

Determine exactly which live `garde` rules and behaviors are already migrated into packages, which remain app-only, and which app behaviors are stale bridges that should disappear in the package model.

## Approach

1. Read every live app `garde` rule body and the app runner/gating code.
2. Read every package `garde` rule body plus package ingestion and public types.
3. Compare app rule semantics to package rule semantics rule by rule.
4. Classify each app behavior as:
   - migrated
   - intentionally replaced
   - app-only and still missing
   - stale bridge that should be removed
5. Report the remaining package work for `garde` under the workspace-local package model.

## Key decisions

- Treat the old app as inventory only, not as a runtime to preserve.
- Use the actual rule bodies and ingestion code as ground truth, not plans or README claims.
- Keep the lane model strict:
  - config checks config contents
  - source checks source files
  - filetree checks file tree
