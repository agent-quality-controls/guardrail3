# Migrate R30-R31 (crate-level allow) from grep to syn

**Date:** 2026-03-16 08:31
**Task:** Rewrite check_crate_level_allow to use syn AST parsing with grep fallback

## Goal
R30/R31 checks use grep-based line scanning. Migrate to syn-based AST inspection via `ast_helpers::find_crate_level_allows`, with grep fallback if parsing fails.

## Approach

### Step-by-step plan
1. Rename current `check_crate_level_allow` to `check_crate_level_allow_grep` (the fallback)
2. Write new `check_crate_level_allow` that:
   - Calls `ast_helpers::parse_file(content)`
   - On success: uses `find_crate_level_allows` to get `(line, lint_name)` pairs
   - For each found allow: looks at the SOURCE TEXT at that line for `//` comment
   - Applies same logic: `unused_crate_dependencies` -> R31 Info, else R30 Error (Info for test files)
   - On parse failure: delegates to `check_crate_level_allow_grep`
3. Update tests to work with new implementation (same function name, same behavior)

### Key decisions
- **Comment detection on source text:** syn strips comments, so we index into `content.lines()` to check for `//` on the same line as the attribute
- **Grep fallback:** keeps backward compatibility for non-parseable files
