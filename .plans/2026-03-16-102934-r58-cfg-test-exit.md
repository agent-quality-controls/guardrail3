# Fix R58 cfg(test) block tracking never exits

**Date:** 2026-03-16 10:29
**Task:** Bug #26 — `in_cfg_test_block` never resets to false

## Goal
After a `#[cfg(test)] mod tests { ... }` block ends, R58 should resume scanning production code.

## Approach

### The bug
In both `check_inline_std_fs_calls` and `check_direct_fs_usage_grep`, the `in_cfg_test_block` flag is set to `true` when entering a cfg(test) block but never set back to `false`. Any production code after the test module is silently skipped.

### The fix
Add brace depth tracking. When `in_cfg_test_block` is true, count `{` and `}` on each line. When depth returns to 0, reset `in_cfg_test_block = false` and `seen_cfg_test = false`.

The brace counting must happen BEFORE the `continue` statement so we actually process closing braces while inside the block.

### Test
Add `r58_catches_production_fs_after_cfg_test` that has a cfg(test) module followed by production code using std::fs.
