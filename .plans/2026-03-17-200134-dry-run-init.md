# Add --dry-run flag to rs init and ts init

**Date:** 2026-03-17 20:01
**Task:** Add `--dry-run` flag to `guardrail3 rs init` and `guardrail3 ts init`

## Goal
Both init commands gain a `--dry-run` flag that shows what files would be created/changed without writing anything.

## Approach
1. cli.rs: Add `dry_run: bool` field to both `RsCommands::Init` and `TsCommands::Init`
2. main.rs: Pass `dry_run` through match arms
3. commands/init.rs: Add `dry_run` param to `run_rs` and `run_ts`, add `show_file_diff` helper, implement dry-run logic

## Files to Modify
- `apps/guardrail3/src/cli.rs` — add dry_run fields
- `apps/guardrail3/src/main.rs` — pass dry_run through
- `apps/guardrail3/src/commands/init.rs` — implement dry-run logic
