## Phase 1: Critical / Bugs (7 issues)

### 1. Workspace root detection — only checks `.` and `apps/backend/`
**File:** `apps/guardrail3/src/app/discover.rs`
**Problem:** `detect_rust()` only checks root and `apps/backend/` for Cargo.toml. Polyglot projects with Rust workspace at `crates/` are completely missed.
**Impact:** 10+ false positives on graf (all config files exist but at `crates/`, not root)
**Fix:** Add `crates/` as a third fallback path in `detect_rust()`. Also consider walking immediate subdirectories for `Cargo.toml` with `[workspace]`.

### 2. R-GARDE-05 fires when garde not in deps
**File:** `apps/guardrail3/src/app/rs/validate/garde_checks.rs`
**Problem:** R-GARDE-05 lists 42 structs missing Validate even when garde is not a dependency. Following the advice would fail to compile.
**Impact:** 42 false violations on any non-garde project
**Fix:** Gate R-GARDE-05 on R-GARDE-01 passing. If garde is not in deps, skip R-GARDE-05 entirely.

### 3. `is_test_path` misses bare `tests.rs`
**File:** `apps/guardrail3/src/app/rs/validate/source_scan.rs:118-124`
**Problem:** Checks `_test.rs`, `_tests.rs`, `/tests/`, `/test/` but not `/tests.rs` (bare, no underscore).
**Impact:** 147 false R44 warnings on graf from `tests.rs` files
**Fix:** Add `|| path_str.ends_with("/tests.rs")` to `is_test_path()`.

### 4. JSON summary counts don't match filtered sections
**File:** `apps/guardrail3/src/report/json.rs`
**Problem:** Summary counts ALL results (including inventory). Sections filter out inventory items. Machine consumers get contradictory data.
**Fix:** Either (a) make summary count only visible items, or (b) add `visible_errors`, `visible_warnings`, `visible_info` alongside total counts.

### 5. #[allow] on expression statements not visited
**File:** `apps/guardrail3/src/app/rs/validate/ast_visitors.rs`
**Problem:** `ItemAllowVisitor` visits items, impl items, trait items, locals, arms — but NOT `Stmt::Expr` or `Stmt::Semi` with attributes.
**Impact:** `#[allow(clippy::as_conversions)]` on expression statements silently missed.
**Fix:** Add `visit_stmt` to `ItemAllowVisitor`.

### 6. `any` detection misses type aliases
**File:** `apps/guardrail3/src/app/ts/validate/ts_code_analysis.rs:100-127`
**Problem:** `collect_any_types` only matches `type_annotation` and `as_expression`. Type aliases (`type X = Record<string, any>`) use `predefined_type` inside `type_arguments` — not matched.
**Fix:** Add match arm for `"predefined_type"` node kind with text `"any"`.

### 7. `any` in nested `as` expressions not caught
**File:** Same as #6
**Problem:** `as_expression` only checks direct `predefined_type` children. `as [string, any][]` has nested `any`.
**Fix:** Use `has_predefined_any_child` (recursive) for `as_expression`.

## Phase 2: High Priority (8 issues)

### 8. ESLint value check flags stricter values as wrong
**File:** `apps/guardrail3/src/app/ts/validate/eslint_check.rs:283+`
**Problem:** String-matches expected value. `complexity: 15` when expected `25` → mismatch. But 15 is STRICTER.
**Fix:** Parse numeric values and compare. Value <= expected → pass. Value > expected → fail.

### 9. TS file length miscounts block comments
**File:** `apps/guardrail3/src/app/ts/validate/source_scan.rs:220-243`
**Problem:** Simple line filter doesn't track block comment state. Lines inside `/* ... */` without leading `*` are counted as code.
**Fix:** Port RS's `filter_non_comment_lines()` block-comment tracking to TS file length check.

### 10. T34/T35 false positive on string literals
**File:** `apps/guardrail3/src/app/ts/validate/source_scan.rs:246-279`
**Problem:** `line.contains(pattern)` matches inside string literals. `"// noinspection"` in a string triggers T34.
**Fix:** Use tree-sitter to check if the match is inside a comment node (not a string). Or strip string content before matching.

### 11. T-ARCH checks only look in `apps/*/src/modules/`
**File:** `apps/guardrail3/src/app/ts/validate/ts_arch_checks.rs`
**Problem:** Misses `packages/{domain,ports,adapters}/` pattern (used by graf).
**Fix:** Add support for multiple hex arch patterns: `apps/*/src/modules/`, `packages/{domain,ports,adapters}/`, `src/{domain,ports,adapters}/`.

### 12. T-TEST-03 only checks root for vitest config
**File:** `apps/guardrail3/src/app/ts/validate/test_checks.rs:123-159`
**Problem:** `path.join(name).exists()` only checks root. Monorepos have per-package configs.
**Fix:** Also search `packages/*/` and `apps/*/` for vitest/jest configs.

### 13. R-BIN-03 fires on publish=false crates
**File:** `apps/guardrail3/src/app/rs/validate/release_bin_checks.rs`
**Problem:** Flags missing binstall metadata on private crates.
**Fix:** Skip crates with `publish = false` in Cargo.toml.

### 14. R-TEST-09 message factually wrong
**File:** `apps/guardrail3/src/app/rs/validate/test_checks.rs`
**Problem:** Claims `#[cfg(test)]` "increases compile time for the main binary" — cfg(test) is stripped in non-test builds.
**Fix:** Remove the compile time claim. Keep the message about separating test code.

### 15. Absolute paths in output
**File:** `apps/guardrail3/src/report/text.rs`, `markdown.rs`
**Problem:** Full absolute paths add noise. `/Users/tartakovsky/Projects/websmasher/guardrail3/apps/guardrail3/src/app/hooks/hook_checks.rs` should be `apps/guardrail3/src/app/hooks/hook_checks.rs`.
**Fix:** Strip `report.project_path` prefix from file paths in text and markdown output. JSON should include both relative and absolute.

## Phase 3: Medium Priority (10 issues)

### 16. TS has no input validation check (Zod)
Add T-ZOD-01 (Zod in deps), T-ZOD-02 (ban raw JSON.parse), T-ZOD-03 (API route validation).
Evidence: 10+ real violations in steady-parent.

### 17. TS has no dependency allowlist
Add T-DEPS-01/02 (equivalent of R-DEPS-01/02 but for package.json).

### 18. RS doesn't load .gitignore
Port gitignore loading from TS source_scan to RS source_scan.

### 19. File length severity inconsistent
RS=Error(>500), TS=Warn(>300). Align severity or document the difference.

### 20. TS doesn't skip test files from source scan
Add `is_ts_test_file()` exclusion in TS source scan, same pattern as RS.

### 21. "--verbose" appears twice in R33 summary
Title says "review with --verbose" and summary appends "(use --verbose to list each)". Remove from title.

### 22. Summarization threshold 3 too low
Change from 3 to 5 or 10 in text.rs/markdown.rs.

### 23. No `--quiet`, `--severity`, `--no-color` flags
Add CLI flags to ValidateArgs. `--quiet` = summary only. `--severity error` = only errors. `--no-color` = disable ANSI.

### 24. No JSON schema version or pass/fail status
Add `"schema_version": 1`, `"passed": false` to JSON output.

### 25. T50 route wrappers fires on non-web projects
Detect project type from package.json dependencies. Skip T50 when no HTTP framework found.
