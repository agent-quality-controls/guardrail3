# T-ARCH checks — TypeScript hex arch enforcement

**Date:** 2026-03-16 16:55
**Task:** Add T-ARCH-01/02 checks for TypeScript services

## Convention
```
apps/my-app/src/modules/
  domain/              pure types, zero imports from application/adapters
  ports/
    inbound/           route/API interfaces
    outbound/          repository/client interfaces
  application/         use cases (NOT "app" — reserved by Next.js)
    commands/
  adapters/
    inbound/           API handlers
    outbound/          DB clients, HTTP clients
```

## Checks

### T-ARCH-01: TS service missing hex arch structure
For each app detected in apps/, check that src/modules/ exists with at least domain/ and adapters/.
Severity: Warn.

### T-ARCH-02: TS import boundary violation
Scan .ts/.tsx files in modules/ for imports that cross layer boundaries:
- domain/ files must NOT import from application/, adapters/, or ports/
- ports/ files must NOT import from application/ or adapters/
- application/ files must NOT import from adapters/
- adapters/ can import from anything

Detection: read import statements, check if the resolved path crosses into a forbidden layer.
Use tree-sitter to parse imports (already have TS AST infrastructure).
