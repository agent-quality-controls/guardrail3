# Scaffold hex arch for cargo-test-reduce-cli

**Date:** 2026-03-16 16:08
**Task:** Full hex arch crate structure inside apps/cargo-test-reduce-cli following the ts-rust-railway template pattern.

## Goal
Replace the flat single-binary CLI with proper hex arch crates. Pure logic stays in packages/cargo-test-reduce. CLI gets domain/ports/app/adapters layers.

## Structure
```
apps/cargo-test-reduce-cli/
  crates/
    domain/types/              ← CoverageMatrix, TestId, LineId, RedundancyScore, errors
    ports/outbound/coverage/   ← trait CoverageProvider (abstracts cargo-llvm-cov)
    app/commands/              ← use cases: reduce command orchestration
    adapters/
      inbound/cli/             ← clap CLI, main.rs (composition root)
      outbound/llvm-cov/       ← concrete: shells out to cargo-llvm-cov, parses JSON
```

## Clippy.toml per layer
- domain/types — strict: no I/O, no process, no global state
- ports/outbound/coverage — strict: no I/O, no process, no global state (trait defs only)
- app/commands — strict: no I/O, no process, no global state
- adapters/inbound/cli — composition root: LazyLock allowed, fs/process NOT banned (wires everything)
- adapters/outbound/llvm-cov — adapter: Command::new allowed, fs allowed (reads coverage JSON), global state banned

## Dependency direction
cli → commands → coverage (port trait) + types + cargo-test-reduce (package)
llvm-cov → coverage (port trait) + types
cli → llvm-cov (wires concrete adapter to port)
