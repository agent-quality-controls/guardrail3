# Goal

Implement a real `TS-STYLE` family for framework-independent CSS and Tailwind policy.

The family must enforce delegated tooling and effective config wiring. It must not become an Astro rule, must not parse Tailwind classes itself, and must not duplicate Stylelint or ESLint behavior.

# Evidence From SteadyParent

SteadyParent has two style enforcement surfaces:

- Root `.stylelintrc.mjs` extends `stylelint-config-standard` and `stylelint-config-tailwindcss`, loads `@double-great/stylelint-a11y`, and enables strict CSS a11y rules.
- Root `eslint.config.mjs` uses `eslint-plugin-tailwind-ban` on `apps/landing/src/**/*.tsx` with a concrete denylist for ad-hoc typography/color/prose Tailwind classes.

Weakness to avoid:

- SteadyParent sets `tailwind-ban/no-deny-tailwind-tokens` to `warn`. G3TS must require `error`.

Current websmasher gap:

- Root ESLint imports `eslint-plugin-tailwind-ban`.
- Global rule has `denyList: []`, which is a no-op.
- Strong denylist exists for `packages/ui` and `apps/web`, but not `apps/landing`.
- Stylelint exists, but no G3TS family proves the required package/config/script contract.

# Architecture

Create package structure:

- `packages/parsers/stylelint-config-parser`
- `packages/ts/style/g3ts-style-types`
- `packages/ts/style/g3ts-style-ingestion`
- `packages/ts/style/g3ts-style-config-checks`
- `packages/ts/style/g3ts-style-hook-contract`

Do not create source checks in this slice. Source misuse is delegated to `eslint-plugin-tailwind-ban`.

# Config Contract

Add `[ts.style]` to `guardrail3-ts.toml`.

Required fields:

- `source_globs`: non-empty app/package-relative source lanes protected by Tailwind policy.
- `tailwind_denylist`: non-empty list of denied Tailwind classes/tokens.
- `stylelint_css_globs`: non-empty CSS globs that must be covered by the CSS lint script.

Path/list rules:

- globs must be relative.
- globs must not contain `..`.
- denylist entries must be non-empty.

# Ingestion

Use existing parsers/support:

- `package-json-parser` for package deps and scripts.
- `package-script-command-parser` for fail-closed script command facts.
- `eslint-config-parser` effective config probes for `tailwind-ban`.
- `stylelint-config-parser` effective config probes through Stylelint's own `resolveConfig` API.

Do not hand-parse Tailwind classes.

Do not hand-parse Stylelint JS config. The parser must execute Stylelint's maintained config resolver from the checked app/package dependency graph and return typed facts.

# G3TS Rules

`g3ts-style/strict-policy-configured`

- Error when `[ts.style]` is missing or any required list is empty.

`g3ts-style/policy-paths-valid`

- Error when configured globs are absolute, empty, or contain `..`.

`g3ts-style/style-packages-present`

- Require direct dependency/devDependency presence:
  - `stylelint`
  - `stylelint-config-standard`
  - `stylelint-config-tailwindcss`
  - `@double-great/stylelint-a11y`
  - `eslint-plugin-tailwind-ban`

`g3ts-style/stylelint-config-present`

- Require a Stylelint config at the app/root being checked.
- Accepted names:
  - `.stylelintrc.mjs`
  - `.stylelintrc.cjs`
  - `.stylelintrc.js`
  - `stylelint.config.mjs`
  - `stylelint.config.cjs`
  - `stylelint.config.js`

`g3ts-style/stylelint-config-stack`

- Require the resolved/raw Stylelint config facts to include:
  - `stylelint-config-standard`
  - `stylelint-config-tailwindcss`
  - `@double-great/stylelint-a11y`

`g3ts-style/stylelint-a11y-rules`

- Require the resolved Stylelint config facts to enable the known strict a11y rules:
  - `a11y/content-property-no-static-value`
  - `a11y/font-size-is-readable`
  - `a11y/line-height-is-vertical-rhythmed`
  - `a11y/media-prefers-reduced-motion`
  - `a11y/no-display-none`
  - `a11y/no-obsolete-attribute`
  - `a11y/no-obsolete-element`
  - `a11y/no-outline-none`
  - `a11y/no-spread-text`
  - `a11y/no-text-align-justify`
  - `a11y/selector-pseudo-class-focus`

`g3ts-style/css-lint-script`

- Require package script `lint:css`.
- It must invoke `stylelint`.
- It must not have parse blockers.
- It must not use `|| true`.
- It must include `--max-warnings 0`.

`g3ts-style/tailwind-ban-eslint-rule`

- Require `eslint-plugin-tailwind-ban` active on every configured `source_globs` probe.
- Require `tailwind-ban/no-deny-tailwind-tokens` at `error`.
- Require rule option `denyList` exactly equals `[ts.style].tailwind_denylist`.

# Runner Wiring

Add `SupportedFamily::Style`.

Run it in the structure runner because this family needs package, file-tree, ESLint, and script facts.

- invalid glob path
- missing package
- missing stylelint config
- config missing Tailwind config
- config missing a11y rule

- `lint:css` with `|| true`

- Tailwind rule at `warn`
- Tailwind rule with empty/wrong denylist
- valid config passes
