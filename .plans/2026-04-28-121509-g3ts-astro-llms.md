# Goal

Create the narrow TypeScript Astro integration package `g3ts-astro-llms` that generates `dist/llms.txt` during Astro builds.

# Approach

- Add `packages/ts/astro/integrations/g3ts-astro-llms` as a standalone npm package following existing local package conventions.
- Export a default Astro integration factory from `src/index.ts`.
- Validate explicit config with Zod: `title`, `site`, `sections`, and optional explicit `contentAdapters`.
- Keep generation deterministic by sorting sections and links, normalizing site URLs, and writing only `llms.txt`.
- Add package-local Node test runner tests for valid generation, validation failures, URL normalization, and non-responsibility boundaries.
- Add README with usage and explicit non-responsibilities.

# Key Decisions

- Use a standalone package-local lockfile because existing TypeScript packages are standalone and there is no root workspace package manager file.
- Do not implement content adapter reading in this pass because the plan only allows it when explicitly configured and the requested scope is narrow `llms.txt` generation.
- Do not touch Rust checker packages or TS checker packages.
