# Goal

Make `g3rs validate --path apps/guardrail3-ts` pass without weakening Rust guardrail semantics.

# Current Failures

- `g3rs-workspace-crawl` is used by the G3TS app, but `apps/guardrail3-ts/guardrail3-rs.toml` only allowlists the workspace alias `g3-workspace-crawl`.
- Astro family crates were added to `guardrail3-ts-family-runner-structure`, but the app-level Rust guardrail dependency allowlist was not updated.
- The same missing allowlist entries also trip app-architecture impure dependency checks, because those checks read `allowed_deps` from the same app config.

# Approach

- Edit only `apps/guardrail3-ts/guardrail3-rs.toml`.
- Add the actual Cargo package name `g3rs-workspace-crawl` alongside the alias already present.
- Add the three Astro family crates used by the structure runner:
  - `g3ts-astro-config-checks`
  - `g3ts-astro-file-tree-checks`
  - `g3ts-astro-ingestion`
- Do not add broad waivers and do not change Rust guardrail code.

# Verification

- `g3rs validate --path apps/guardrail3-ts`
- `cargo test --workspace` in `apps/guardrail3-ts`
