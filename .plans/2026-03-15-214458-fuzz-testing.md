# Set up fuzz testing for guardrail3

**Date:** 2026-03-15 21:44
**Task:** Set up cargo-fuzz with targets for config parsing, source scanning, and project detection.

## Goal
Fuzz testing infrastructure that exercises the main input-processing functions with random data to find panics and logic errors.

## Input Information
- guardrail3 is binary-only (no lib.rs) — need to add a lib target for fuzz imports
- Key fuzzable surfaces:
  1. TOML config parsing (`toml::from_str::<GuardrailConfig>`)
  2. Source scan comment filtering (`filter_non_comment_lines`)
  3. Allow/structure/code-quality checks (operate on string content)
  4. Project detection (needs filesystem, harder to fuzz directly)
- cargo-fuzz 0.13.1 is installed
- Crate uses edition 2024, rust-version 1.85
- Strict clippy: no unwrap, no expect, no unsafe, etc.

## Approach

### Step-by-step plan
1. **Add `src/lib.rs`** — re-export the modules fuzz targets need. The binary crate stays as-is via `[[bin]]`. Add `[lib]` section to Cargo.toml.
2. **Create `fuzz/Cargo.toml`** — standard cargo-fuzz manifest depending on the guardrail3 lib.
3. **Create fuzz targets:**
   - `fuzz_config_parse.rs` — feed arbitrary strings to `toml::from_str::<GuardrailConfig>`. Tests that deserialization never panics.
   - `fuzz_source_scan.rs` — feed arbitrary strings to `filter_non_comment_lines`, `check_item_level_allow`, `check_file_length`, `check_unsafe`, `check_todo_macros`, `check_unwrap_expect`, `check_direct_fs_usage`, `check_garde_skip`, `check_cfg_attr_allow`, `check_crate_level_allow`. Tests that source scanning never panics on any input.
   - `fuzz_discover.rs` — write random Cargo.toml content to temp dir, run `detect_project`. Tests project detection with malformed TOML.

### Key decisions
- **Adding lib.rs:** Required because fuzz targets compile as separate crates and need to import guardrail3 functions. The lib.rs will only re-export what's needed.
- **Making modules pub:** Some validate submodules are private (allow_checks, structure_checks, code_quality_checks). They need to be `pub` (or `pub(crate)` won't work across crate boundary). Will make them `pub` in rs/validate/mod.rs.
- **No binary invocation in fuzz:** Too slow. Direct function calls.
