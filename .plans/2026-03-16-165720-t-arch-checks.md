# Add T-ARCH-01 and T-ARCH-02 for TypeScript hex arch enforcement

**Date:** 2026-03-16 16:57
**Task:** Implement two new TS architecture checks

## Goal
Add T-ARCH-01 (missing hex arch structure) and T-ARCH-02 (import boundary violation) checks for TypeScript apps.

## Approach

### Step-by-step plan
1. Create `src/app/ts/validate/ts_arch_checks.rs` with both checks
2. Add `pub mod ts_arch_checks;` to `src/app/ts/validate/mod.rs`
3. Wire checks into the architecture section of the TS validate orchestrator
4. Update `src/help_gen.rs` — add T-ARCH-01/02 under ARCHITECTURE section in TS_VALIDATE_HELP

### Key decisions
- **Detection for T-ARCH-01:** Use `fs.list_dir()` to scan `apps/` for subdirs that have `package.json` or `src/`. Check that `src/modules/domain/` and `src/modules/adapters/` exist.
- **Detection for T-ARCH-02:** Simple line-by-line scanning for import/require statements. No AST needed — just look for `from '`/`from "` and `require(` patterns, then check resolved paths against forbidden layer rules.
- **StubFs:** Copy the pattern from hex_arch_checks.rs tests. StubFs only supports `read_file` (not directory listing), so for T-ARCH-01 we'll check for known marker files rather than listing dirs.
