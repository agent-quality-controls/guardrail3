# Add golden file testing for guardrail3

**Date:** 2026-03-15 21:29
**Task:** Add golden file testing that snapshots `guardrail3 validate --format json` output and detects behavioral changes.

## Goal
Self-validation golden tests: run `guardrail3 validate` against its own repo, snapshot the JSON output, and fail CI if output changes unexpectedly.

## Approach

### Step-by-step plan
1. Create `golden-tests/run-golden.sh` — builds release, runs self-validation, normalizes JSON, saves golden snapshot
2. Create `golden-tests/compare.sh` — builds, runs, normalizes, diffs against golden snapshot
3. Normalization: use `jq` to sort checks by ID, strip absolute paths (replace with `.`), remove line numbers (nondeterministic for some checks), keep severity/id/title/message
4. Capture initial golden snapshot by running `run-golden.sh`
5. Note CI integration as TODO (no `.github/workflows/ci.yml` exists)

### Key decisions
- **Use jq for normalization:** Portable, standard, handles JSON sorting deterministically
- **Strip file paths to relative:** Absolute paths change per machine
- **Keep line numbers:** They're part of the check output and should be stable for self-validation (source files don't change without intent)
- **POSIX sh where possible:** Scripts use `#!/bin/sh` but depend on `jq` and `cargo`

## Files to Create
- `golden-tests/run-golden.sh` — capture golden snapshot
- `golden-tests/compare.sh` — compare current output against golden
- `golden-tests/golden/self-validate.json` — initial golden snapshot
