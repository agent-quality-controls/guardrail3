# Goal

Forbid the bare `astro-seo` package in Astro apps because `astro-seo@1.1.0` exports TypeScript source directly and is not part of the approved delegated SEO pipeline.

# Approach

- Add `astro-seo` to the Astro Syncpack forbidden dependency contract in `g3ts-astro-ingestion`.
- Update config-check test helpers so golden fixtures require the same ban.
- Improve `TS-ASTRO-CONFIG-10` so the error message explains why `astro-seo` is forbidden and what to use instead.
- Add a regression test that proves missing the `astro-seo` ban reports the package explicitly.
- Run the Astro ingestion/config-check suites, the G3TS CLI suite, formatting, install the CLI, and validate the landing app reports the new actionable error.

# Key Decisions

- Do not re-add `astro-seo` as a required package. The approved stack is `schema-dts` for typed JSON-LD, Nuasite rendered-output checks, `@astrojs/sitemap`, and `astro-robots`.
- Keep banning `astro-seo-meta` and `astro-seo-schema`; add bare `astro-seo` to the same Syncpack policy.
- Do not publish npm plugins. This is Rust G3TS CLI policy, not ESLint plugin behavior.
