# Family Plan Index

This directory is the current family-level planning surface.

Use it to answer:

- what families exist
- which documents are current
- where the code lives
- what is implemented versus still planned
- which older plan files are now historical or tactical only

Authority order:

1. live code
2. family-local READMEs and shared architecture READMEs
3. this `by_family/` tree for family-level planning/status and reconciliation
4. tactical handoffs / attack plans / migration ledgers
5. historical research and legacy notes

Current subtrees:

- `rs/`
  - current Rust family planning/status
- `ts/`
  - reserved for the TypeScript family cutover once that consolidation starts

This tree does not replace code or family/shared READMEs for behavior.
It replaces the older spread of competing family plan files as the primary planning index.
