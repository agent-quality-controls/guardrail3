# Fix R44 and R58 to respect #[allow] attributes

**Date:** 2026-03-17 10:41
**Task:** When AST scanner finds .unwrap() (R44) or std::fs (R58), skip if enclosing function/module has a relevant #[allow] attribute. Also fix ItemAllowVisitor to catch #[allow] on expression statements.

## Goal
- R44 (unwrap/expect) skips findings when enclosing fn has `#[allow(clippy::unwrap_used)]` or `#[allow(clippy::expect_used)]`
- R58 (std::fs) skips findings when enclosing fn/module has `#[allow(clippy::disallowed_methods)]`
- ItemAllowVisitor also visits expression statements with `#[allow]` attrs (visit_expr_attrs or visit_stmt)
- R33 still reports the allow as audit trail (no change needed there)

## Approach

### Step-by-step plan

1. **Add `has_allow_lint` helper** to `ast_visitors.rs` — checks if an attribute is `#[allow(clippy::X)]` for a given lint name
2. **Modify `UnwrapExpectVisitor`** — add `in_allowed_scope: bool` field, override `visit_item_fn` and `visit_impl_item_fn` to set it, skip in `visit_expr_method_call` when true
3. **Modify `InlineStdFsVisitor`** — add `in_allowed_disallowed_methods: bool` field, override `visit_item_fn` and `visit_impl_item_fn` to check for `#[allow(clippy::disallowed_methods)]`, skip in visit_expr_call/visit_expr_path when true
4. **Modify `ItemAllowVisitor`** — add `visit_stmt` to catch `#[allow]` on expression statements (Stmt::Expr with attrs via the Expr's attrs)
5. **Update `find_unwrap_expect` in ast_helpers.rs** to initialize new field
6. **Add tests** for each change
7. **Run cargo test + clippy**

### Key decisions
- Track scope via bool field with save/restore pattern (same as `in_cfg_test` already used in `InlineStdFsVisitor`)
- Check both `visit_item_fn` and `visit_impl_item_fn` for completeness
