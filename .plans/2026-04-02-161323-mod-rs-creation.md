# Create mod.rs for rule-group directories (cargo, hexarch, deny families)

## Goal
Replace `#[path = "dir/file.rs"] mod file;` patterns in lib.rs with standard Rust module resolution via `mod.rs` files.

## Directories to handle

### Cargo (2 dirs, 15 rules)
- `member_policy/` (6 rules)
- `workspace_policy/` (9 rules)

### Hexarch (4 dirs, 22 rules)
- `structure/` (6 rules)
- `workspace_policy/` (4 rules)
- `dependency_policy/` (6 rules)
- `dependency_integrity/` (9 rules)

### Deny (5 dirs, 25 rules)
- `advisories/` (5 rules)
- `bans/` (9 rules)
- `coverage/` (2 rules: 01, 03)
- `licenses/` (4 rules)
- `sources/` (7 rules)

## Changes per directory

1. **Create `dir/mod.rs`**: Declares each `.rs` file as `pub(crate) mod name;` plus test sidecar dirs with `#[cfg(test)]`
2. **Update rule `.rs` files**: Change `super::X` to `crate::X` in use statements (since `super` now points to mod.rs, not lib.rs)
3. **Update parent `lib.rs`**: 
   - Remove all `#[path = "dir/..."] mod ...;` lines
   - Add `mod dir;` for each directory
   - Update `check()` function: `rule_name::check(...)` becomes `dir::rule_name::check(...)`
   - Update any `use self::` references if they reference moved modules

## Key decisions

- Use `crate::` instead of re-exports in mod.rs (cleaner, no coupling)
- Test sidecar `super::super::` references stay unchanged (they point to the rule file, not lib.rs)
- Test sidecar dirs already have their own mod.rs; they don't need entries in the new directory mod.rs since they're declared inside their parent rule file
- `pub(crate) mod` in the new mod.rs (since lib.rs used `mod`, we need pub(crate) so lib.rs can still use `dir::rule::check()`)

## Execution order
1. Cargo family (simplest, verify pattern works)
2. Hexarch family
3. Deny family
4. `cargo check` after each family
