Goal

Audit the `topology` family against exact old app rule code and exact package code. Determine which old rules are still meaningful under the pointed-workspace package model, which are migrated, which are intentionally out of scope, and which public package contracts are still wrong.

Approach

1. Read the old app `topology` README and all live rule files to reconstruct the original repo-global intent and rule inventory.
2. Read the package `topology` types, ingestion, and file-tree checks to reconstruct the current pointed-workspace package contract.
3. Run focused adversarial reviewers on:
   - old app inventory and semantics
   - package file-tree implementation and tests
   - migration fit from repo-global to pointed-workspace
   - public contract and lane honesty
4. Consolidate the findings into:
   - migrated
   - not migrated but still meaningful
   - intentionally not part of the package model
   - package contract bugs

Key decisions

- Judge old app rules by merits under the current package model, not by parity alone.
- Treat fake public lanes in `topology` as a real contract defect.
- Separate "repo-global old app rule" from "workspace-local package successor" explicitly.
