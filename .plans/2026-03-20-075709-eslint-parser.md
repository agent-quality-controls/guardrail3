# Create tree-sitter-javascript ESLint config parser

**Date:** 2026-03-20 07:57
**Task:** Add tree-sitter-javascript dependency and create ESLint flat config parser

## Goal
Create a structured ESLint config parser using tree-sitter-javascript that extracts rules, presets, plugin detection, and marker detection from `.mjs` files.

## Approach

### Step-by-step plan
1. Add `tree-sitter-javascript = "0.25"` to Cargo.toml dependencies
2. Create `eslint_parser.rs` with `EslintConfig` and `RuleConfig` structs
3. Implement tree-sitter-javascript parsing of ESLint flat config
4. Add `mod eslint_parser;` to `mod.rs`
5. Verify compilation with `cargo check`

### Key decisions
- **tree-sitter-javascript for .mjs parsing:** ESLint flat config is JavaScript, not TypeScript. The existing tree-sitter-typescript is for TS source files.
- **BTreeMap for rules:** Consistent with codebase convention (HashMap is banned)
- **Last-occurrence-wins for rules:** Later config blocks override earlier ones in ESLint flat config
