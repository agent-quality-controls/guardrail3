# Switch release trigger from main to production branch

**Date:** 2026-03-15 22:10
**Task:** Agents push to main freely. Releases only on merge to production.

## Approach
- Create `production` branch
- release.yml triggers on push to `production`, not `main`
- binary-release.yml stays on tag push (unchanged)
- CI stays on main + PRs (unchanged)
- Deploy flow: main → PR to production → merge → release-plz → crates.io + tag → binaries
